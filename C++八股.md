# `new`和`malloc`有什么区别
`new`和`malloc`都是用于动态内存分配，它们的区别有
1. 语法层面
	- `new`是`C++`操作符，它的行为是由编译器实现的
	- `malloc`是C语言中的一个函数，需要包括头文件`<cstdlib>`
2. 返回类型
	- `new`是类型安全的，它首先会根据变量类型分配内存，然后调用构造函数初始化对象，并返回指定类型指针。
	- `malloc`不是类型安全的，它只分配内存，需要指明要分配的内存空间大小，且不会调用构造函数，返回的类型是`void*`，需要强制转换为具体的指针类型，再进行手动初始化
3. 构造和析构
	- `new`分配的对象在对象生命周期结束时，需使用`delete`自动调用对象的析构函数
	- `malloc`分配的内存不会通过`free`自动调用析构函数，所以如果创建了对象，必须先手动调用析构函数，再用`free`释放内存
4. 异常安全性
	- `new`在分配失败时会抛出`std::bad_alloc`的异常
	- `malloc`在分配失败时，返回`NULL`
# 堆和栈的区别
## 栈
栈是由**编译器**自动管理的，用于存储局部变量和函数调用的上下文；
栈上对象在定义他们的作用域结束后**自动销毁**
栈在虚拟地址空间中通常表现为一段规整的连续区域，内存分配和释放通过移动栈指针完成，开销小，所以内存分配和释放速度很快。
栈的大小有限制，不同系统和编译器可能不同
## 堆
堆是由程序员手动管理的，用于存储动态分配的对象。
堆上对象需要程序员手动释放，否则可能导致内存泄漏。
堆的内存分配和释放速度通常比栈慢，因为要在一块大的内存空间搜索合适的内存块，不同块之间不保证连续或规整，容易产生内存碎片。
堆的大小通常很大，受限于系统可用内存。

# 什么是内存泄漏, 如何检测和防止？
- 内存泄漏是指程序员在内存池中动态申请了一块内存而没有正确释放，导致这块内存在程序结束之前一直被占用，造成了资源的浪费。
- 内存泄漏可以分为如下几种情况
	  1. 第一种，**忘记手动释放内存**。使用`new`/`malloc`手动分配内存后，没有使用`delete/free`手都释放，导致了堆区内存泄露。
	  2. 第二种是，没有设置基类的析构函数为虚函数。这种情况对应于，当我希望通过基类指针或引用销毁多态的派生类对象时，必须将基类的析构函数设置为虚函数。否则，当我们`delete`的时候，之后调用基类的析构函数，而不会调用派生类的析构，造成内存泄漏。
	  3. 第三种是野指针。野指针是指指针指向的内存已经被释放掉了。
	  4. 第四种是循环引用。当两个对象互相持有对方的智能指针的时候，导致引用计数无法归零，内存无法释放。
	  5. 第五种是释放函数与分配方式不匹配。比如`new[]`用于分配对象数组只能用`delete[]`释放。
- 如何避免内存泄漏呢？
	1. 使用**RAII资源获取即初始化的思想**，释放动作和对象的生命周期绑定，智能指针是RAII的一种实例，比如说unique_ptr，当make这个对象时，就创建资源，当离开作用域的时候，unique_ptr的资源也自动释放了。
	2. 确保手动分配的内存在不需要的时候也要手动释放
	3. 使用内存泄漏检测工具，如`AddressSanitizer`
	
# 循环引用的例子
循环引用是指多个对象互相持有`shared_ptr`，导致引用计数始终无法将为0，从而使对象无法释放，造成内存泄漏。
最典型的情况是两个对象互相用`shared_ptr`指向对方。解决方法通常是将其中一方改为`weak_ptr`，因为`weak_ptr`只观察对象，不增加引用计数。
```
#include <iostream>
#include <memory>
using namespace std;

struct B;  // 前向声明

struct A {
    shared_ptr<B> b_ptr;   // A 持有 B
    ~A() { cout << "~A() called\n"; }
};

struct B {
    shared_ptr<A> a_ptr;   // B 持有 A
    ~B() { cout << "~B() called\n"; }
};

int main() {
    auto a = make_shared<A>();
    auto b = make_shared<B>();

    a->b_ptr = b;   // A 引用 B
    b->a_ptr = a;   // B 引用 A

    // 程序结束时，a 和 b 离开作用域
    // 但 ~A() 和 ~B() 都不会被调用
    return 0;
}

```
# 什么是野指针？如何避免
野指针就是指指向未知内存的指针，这个未知内存可能是已经释放的、未被初始化的、不确定的、非法的。在C/C++这种允许操作内存地址的语言中，使用野指针可能会导致程序崩溃等其它不可预测的错误。那么什么情况下会产生野指针呢？
`delete`释放内存后，没有对指针置空，指针仍然指向已经被释放内存的地址。所以在释放内存后要将指针置空。
避免返回局部变量的指针。如果函数返回指向其局部变量的指针，一旦函数返回，这些局部变量的生命周期也随之结束，返回的指针成为野指针。
越界访问，指针访问的内存超出了其合法的内存边界，我们可以用安全的迭代风格来操作指针。
函数参数的指针在函数内被错误释放，所以我要注意参数的生命周期。避免释放函数输入的对应内存空间。
同时，使用智能指针也能有效预防野指针。

# C++面向对象三大特性
面向对象编程是C++的核心特性之一，OOP是一种编程范式，程序可以以对象的形式组织数据和功能，并通过对象之间的交互实现任务。面向对象编程具有三大特性，封装、继承和多态。
封装：就是通过把客观事物封装成抽象的类中，对内隐藏细节，对外暴露接口，防止外界干扰或不确定性访问。
继承：是指一个类可以从另一个类中继承属性和行为的能力。派生类可以复用基类的代码，并能在不修改基类的前提下，添加或者修改已有功能。继承方式有公有、私有和受保护，分别决定派生类对基类成员的访问权限。
多态：指同一接口在不同对象中有不同表现形式。C++中，多态通过虚函数来实现。在基类中定义虚函数，派生类中重写该函数，实现运行时的动态绑定。

# 简述一下 C++的重载和重写，以及它们的区别和实现方式
重载和重写都是用于实现多态性，但方式和使用场景有所不同。
首先说重载。重载是编译期实现的静态多态。他是指在同一个作用域内，可以声明参数列表不同的多个同名函数。这里的参数列表不同指的是参数个数不同或者参数类型不同或者参数顺序不同，但是返回值不同不能单独作为重载函数。
重写是发生在继承关系中，重写是在运行期实现的动态多态。基类的函数必须是虚函数，派生类在重写的时候需要声明override关键字，且函数名、参数列表、返回类型必须和基类一致，或者满足协变返回类型。

# 介绍一下C++的锁
在C++11之后，标准库提供了`std::mutex`来实现互斥锁，它能保证同一时间只有一个线程访问共享资源。`std::mutex`有三个主要函数：
`lock`用于阻塞获取锁，如果锁被占用，当前线程会阻塞等待
`try_lock`是非阻塞获取获取锁，即尝试获取锁并立即返回布尔值。
`unlock`解锁，用于释放互斥量。
锁所保护的代码叫作临界区，必须保证在离开临界区后，调用`unlock`释放锁，否则容易产生死锁。
而在实际开发中，单独使用`mutex`在函数返回抛出异常等情况下，容易出现未释放互斥量的问题。为了避免这种情况，通常使用RAII封装，即构造时加锁，析构时自动解锁。具体实现为：`lock_guard`，它的内部持有`mutex`引用，构造函数自动调用`lock()`，析构函数调用`unlock`。写法简洁，作用域结束自动解锁，避免忘记释放导致死锁。
`unique_lock`比`lock_guard`更加灵活，提供了额外功能。如：延迟加锁、手动`unlock/lock`多次等功能。

# 介绍一下条件变量
条件变量时在并发程序中经常用到的一种同步原语，它可用于在线程间**发送通知**，以实现线程间同步。在使用条件变量时，接收线程需要一个条件变量来等待来自发送线程的通知。直到收到这个通知后，才可以继续后面的工作。
以接收线程和发送线程为例。接受线程需要等待发送线程写入数据后才能读取数据。在程序开始运行时，接受线程先获取锁，声明`condition_variable`条件变量等待发送线程的通知。在进入等待状态的同时，释放锁。发送线程获取锁，写入数据，完成写入数据后，发送线程释放锁并使用条件变量发送通知。接收线程收到条件变量通知后，重新获取锁，结束阻塞状态，开始读取和处理共享数据。完成一次通信过程。
条件变量中主要提供了两类函数，等待和通知。以`wait`函数为例，它需要传入`unique_lock`的引用，`wait()`函数内部实现 释放锁 + 阻塞等待 两个操作。当另外一个线程调用`notify`函数发送通知后，`wait()`函数就会结束阻塞，同时会再次获取这个互斥量。
而实际应用中可能会出现丢失唤醒和虚假唤醒的情况。所以可以选用`wait()`函数的第二重写版本，加上一个条件判断参数`void wait(std::unique_lock<std::mutex>& lock, Predicate pred)`，当满足条件，就不会进入下面的等待，相当于一个`while (pred)`循环，如果满足条件才会调用`wait()`函数。这个条件的具体逻辑可以通过`lambda`表达式实现。

## 讲解丢失唤醒和虚假唤醒
丢失唤醒：写线程在读线程进入等待之前就已经准备好了数据，并发送通知；而当读线程进入等待状态后，才准备接受通知，所以就错过了这个通知。
虚假唤醒：等待线程在条件未满足的情况下，就被唤醒了。

# `lambda`和`bind`的区别
`lambda`和`bind`本质都是可调用对象；**优先使用`lambda`**
从语法上来看，`lambda`表达式用来定义一个匿名函数对象，显式展示捕获列表，语义直观，参数传递清晰；`bind`是预先把可调用对象与部分实参绑定在一起，返回一个新的可调用对象；
从性能上来看，`lambda`的函数调用可被编译器内联优化，直接使用原函数；而`bind`是通过函数指针调用。
`lambda`表达式引用捕获需要注意数据竞争和悬挂引用问题。若`lambda`被多个线程调用，且通过引用修改捕获变量，需手动同步；还有就是悬挂引用问题，要注意捕获变量的生命周期。

# 可调用对象都有什么？
凡是可以用函数调用语法`()`调用的对象，统称可调用对象。包含：
- 普通函数
- 类成员函数（静态的类成员函数可以视为普通函数）
- 函数对象
- `lambda`表达式
- `std::bind`
补充：`std::function`是对可调用目标的包装器，通过类型擦除技术实现，符合RAII原则


# C++怎么实现多态
C++的多态分为**编译时多态（静态多态）和运行时多态（动态多态）**。
编译时多态是在编译阶段就能确定具体函数或类型，由编译器根据函数的签名或模板实例化来选择正确的函数或实例。
编译时多态包含，一个是函数重载，是指同一个作用域内可以声明多个同名函数，但他们的参数列表不同。另一个是运算符重载，它允许对用户自定义类型赋予运算符新的语义。例如在类中重载加号运算符，可以实现自定义类的相加。最后一个是模板，它提供了泛型编程的能力，例如我们使用的容器，支持各种各样的数据类型，这本质上也是多态。
运行时多态主要通过虚函数和抽象类实现。其中虚函数是在父类中定义声明，子类中重写。由虚函数表和虚函数指针在运行时确定调用哪个函数。抽象类是指至少包含一个纯虚函数的类，不能被实例化，但可以作为其他类的基类。
具体来说，当类中包含虚函数时，编译器会自动为该创建一个虚函数表，表中包含类中所有虚函数的地址。子类的虚函数表继承了父类的虚函数表，但会使用自己重写的虚函数将虚函数表中对应的虚函数进行覆盖。当通过基类指针或引用调用虚函数时，程序会根据对象的实际类型在运行时查找正确的函数地址并调用相应的函数，实现多态。
## 二刷改进
C++的多态分为编译时多态（静态多态）和运行时多态（动态多态）。
一、编译时多态（静态多态）
编译时多态是指在编译阶段就已经确定调用哪个具体函数。常见形式有：
1. 函数重载：指同一个作用域内，允许声明多个同名函数，但它们的参数列表不同。
2. 运算符重载：C++允许程序员为自定义类型重新定义运算符行为。
3. 模板：允许写一份通用代码，编译器根据具体类型生成不同的版本。
二、运行时多态（动态多态）
运行时多态主要通过继承和虚函数实现。
基类中将成员函数声明为虚函数，子类对其重写，然后通过基类指针或引用调用该虚函数，程序会在运行时根据对象的实际类型选择对应实现。
底层实现的过程为：当一个类里有虚函数时，编译器通常会为该类自动生成一张虚函数表。表中存放这个类虚函数对应的函数地址。如果子类重写了基类的虚函数，那么子类虚函数表中对应位置会被替换为子类自己的函数地址。这样，通过基类指针或引用调用虚函数时，程序会根据对象的虚函数表指针找到所属类的虚函数表，调用对应函数，从而实现多态。

具体例子
```cpp
	#include <iostream>
	using namespace std;
	
	class Animal {
	public:
	    virtual void speak() {
	        cout << "Animal speak" << endl;
	    }
	};
	
	class Dog : public Animal {
	public:
	    void speak() override {
	        cout << "Dog wang wang" << endl;
	    }
	};
	
	class Cat : public Animal {
	public:
	    void speak() override {
	        cout << "Cat miao miao" << endl;
	    }
	};
	
	int main() {
	    Animal* p1 = new Dog();
	    Animal* p2 = new Cat();
	
	    p1->speak();
	    p2->speak();
	
	    delete p1;
	    delete p2;
	    return 0;
	}
```
# 虚函数和纯虚函数的区别
虚函数和纯虚函数都用于实现多态。
虚函数在基类中用`virtual`关键字声明，在派生类中重写。虚函数可以有实现，也就是说，基类中的虚函数可以有定义，派生类可以选择不提供自己的实现，直接使用基类的实现。
纯虚函数是在虚函数后面加一个`=0`，它在基类中只有声明，没有实现。当一类包含至少一个纯虚函数时，它就成为了一个抽象类。抽象类不能被实例化，它的派生类必须提供具体的实现。
所以他们的应用场景有所区别：虚函数用于提供一个可以在派生类中重写的方法实现；而纯虚函数是为了实现抽象类，提供一个中接口规范，要求派生类必须提供具体实现。

# 智能指针
`C++`的智能指针是RAII资源获取即初始化的一种实现。智能指针是标准库提供的一组模板类，用来自动管理动态分配对象的生命周期，它维护着一个指向动态分配对象的指针，并在适当时机自动释放资源，从而减少手动管理内存带来的内存泄漏和悬空指针问题。
`C++`有以下三种智能指针：
1. `std::unique_ptr`：表示独占所有权，不可拷贝，没有引用计数，开销最小。它保证同一时间只有一个`unique_ptr`指向特定内存，适用于不需要共享所有权的场景。
   它的工作原理是：通过**删除拷贝构造函数和拷贝赋值运算符**来确保所有权的唯一性，提供**移动构造函数和移动赋值运算符**，允许所有权的转移。当`unique_ptr`超出作用域时，会自动释放所管理的资源，并调用对象析构函数。
2. `std::shared_ptr`：表示共享所有权，内部维护引用计数，引用计数归零时才会释放资源。多个`shared_ptr`共享同一块内存，适用于多个对象共享同一个资源的场景。
   它的工作原理是：使用**引用计数机制**跟踪有多少个`share_ptr`共享同一资源。内部维护一个控制块，其中包含引用计数和资源的原始指针。每当创建一个新的`shared_ptr`或者将一个`shared_ptr`赋值给另一个时，引用计数加一。当`shared_ptr`被销毁或通过`reset`成员函数重置时，引用计数减少，当引用计数降到0时，控制块会释放资源并自我销毁。
3. `std::weak_ptr`：是弱引用，不拥有对象，不增加`shared_ptr`的计数，只用于观察`shared_ptr`所管理的对象是否存在。它用于解决`std::shared_ptr`之间可能产生的循环引用问题。

口语回答中的小`tips`：指向内存 = 拥有资源；指向特定内存 = 拥有某块资源；内存=资源；指针的销毁 对应于 释放资源

# 智能指针为什么避免传入裸指针？
裸指针：是一个普通原始指针，只保存一个对象地址，没有所有权管理能力。如`int* p; A* p; char* p`都是裸指针。
裸指针只能表示“指向一个对象的地址”，无法表达清楚这块资源到底由谁负责释放，容易导致内存泄漏、重复释放和悬空指针等问题。
最经典的内存泄漏例子就是：函数中途返回，`delete`执行不到
```cpp
void func() {
    A* p = new A();
    if (something_wrong()) return;
    delete p;
}
```

# 左值引用和右值引用的区别
1. 左值和右值
	- 左值是==有名字、能持续存在、可以取地址==的对象。
	- 右值是临时值，没有名字，也不能取地址。
2. 左值引用和右值引用概述
	- 左值引用：是对左值的引用，常用于传递函数参数，避免拷贝大对象，提高效率。
	- 右值引用：是对右值的引用，用于引用临时的或即将被销毁的对象，主要用来实现移动语义。
3. 左值引用
	- 左值引用使用一个`&`符号
	- 左值引用只能引用左值，但是`const`左值引用既可以引用左值，也可以引用右值
		```cpp
		// 1.左值引用只能引用左值
		int t = 8;
		int& rt1 = t;
		
		//int& rt2 = 8;  // 编译报错，因为8是右值，不能直接引用右值
		
		
		// 2.但是const左值引用既可以引用左值
		const int& rt3 = t;
		
		const int& rt4 = 8;  // 也可以引用右值
		const double& r1 = x + y;
		const double& r2 = fmin(x, y);

		```
5. 右值引用
	- 右值引用使用两个`&&`符号
	- 右值引用只能引用右值，但是可以引用被`move`的左值。`move`的作用是将一个左值强制转化成右值，以实现移动语义。
		```cpp
		// 1.右值引用只能引用右值
		int&& rr1 = 10;
		double&& rr2 = x + y;
		const double&& rr3 = x + y;
		
		int t = 10;
		//int&& rrt = t;  // 编译报错，不能直接引用左值
		
		
		// 2.但是右值引用可以引用被move的左值
		int&& rrt = std::move(t);
		int*&& rr4 = std::move(p);
		int&& rr5 = std::move(*p);
		const int&& rr6 = std::move(b);

		```

# 虚函数表（`vtable`)是什么？
1. 虚函数表：虚函数表是编译器为含有虚函数的类生成的一张函数地址表，内部存储了类中所有虚函数的地址。
2. 作用：支持动态绑定，实现运行时多态。
3. 虚函数表指针：虚函数表指针是编译器为含虚函数类的对象添加的一个隐藏成员，它指向这个对象所属类的虚函数表。当通过基类指针或引用调用虚函数时，程序会使用对象的虚函数表指针来查找并调用正确的函数实现。
4. 虚函数表创建时机：虚函数表通常在==编译阶段==由编译器为含有虚函数的类生成。

# 一个类能否有多个虚函数表指针`(vptr)`
能。在简单继承场景下，一个虚函数的对象内部通常只有一个虚函数表指针`vptr`。对于多继承或虚继承的场景下，派生类对象内部会包含多个基类子对象，而每个多态基类子对象都可能需要自己的虚函数表指针，以支持通过不同基类指针进行动态绑定。
举例如下：
```cpp
	class A {
	public:
	    virtual void f() {}
	    int a;
	};
	
	class B {
	public:
	    virtual void g() {}
	    int b;
	};
	
	class C : public A, public B {
	public:
	    void f() override {}
	    void g() override {}
	    int c;
	};
```
则`C c`对象内部如下：
```
	[C 对象内存]
	+-------------------+
	| A部分的 vptr      |
	+-------------------+
	| A::a              |
	+-------------------+
	| B部分的 vptr      |
	+-------------------+
	| B::b              |
	+-------------------+
	| C::c              |
	+-------------------+
```

# 什么是构造函数和析构函数？可以是虚函数吗？
1. 构造函数：
	- 构造函数是创建对象时自动调用的成员函数，它的作用是初始化成员变量，为对象分配资源。
	- 特点：
		（1）函数名与类名相同，且没有返回类型
		（2）可以有多个构造函数
		（3）如果没有为类定义一个构造函数，编译器会自动生成一个默认构造函数。
	- 构造函数不能为虚函数。虚函数调用依赖对象内部的虚函数表指针（`vptr`）来完成动态绑定。如果此时构造函数是虚函数，对象还没实例化，没有分配内存空间，也就无法调用。构造函数也没有必要是虚函数，这是因为虚函数依赖对象构造完成后的多态机制，而构造函数本身就是在创建对象，不适合通过动态绑定来调用，而且构造函数的调用对象在编译阶段就已经确定，不需要虚函数机制。
2. 析构函数
	- 析构函数是对象生命周期结束时自动调用的成员函数，它的作用是释放对象占用的资源，执行一些必要的清理操作。
	- 特点：
		（1）函数名为`~类名`，没有返回类型
		（2）没有参数
		（3）如果没有为类定义一个析构函数，编译器会自动生成一个默认析构函数。
	- 析构函数可以是虚函数。当一个类作为多态基类使用时，通常应该把析构函数声明为虚函数。这是因为如果基类的析构函数不是虚函数，当通过基类指针去删除派生类对象时，不会调用派生类的析构函数，从而会导致派生类的资源未被正确释放，造成内存泄漏。

# `extern`和`extern "C"`的作用
1. `extern`
	- 主要作用：声明变量或函数（声明不同于初始化，故不分配内存），告诉编译器这个变量或函数是在其它文件中定义的，这样链接器在链接阶段会去其他文件寻找定义。
2. `extern "C"`
	- 主要作用：按`C`语言的链接规则来处理后面的函数或变量，而不是按`C++`的方式做名字修饰。这是因为`C++`支持函数重载，`C++`编译器会对函数名字做名字修饰。

# `std::move`和`std::forward`
1. `std::move`：`std::move`的作用是把一个对象无条件转换成右值引用。用于移动语义，从而实现资源转移。
2. `std::forward`：`std::forward`的作用是在模板中，把参数按照它原来传递进来的左值/右值属性，继续传递出去。用于完美转发，使目标函数接收到的实参与被传递给转发函数的实参值类别保持一致。
3. 完美转发：完美转发通过`T&&`转发引用（也叫万能引用）、模板类型推导、引用折叠和`std::forward<T>(arg)`来实现的，从而同时正确处理左值和右值。
4. 引用折叠：将引用的引用转换为单个引用，规则是只有右值引用的右值引用会被折叠成右值引用，剩下的三种情况出现左值引用就会被折叠为左值引用。
5. 注：
	（1）万能引用 = 右值引用 + 模板；如`T&& arg`是万能引用，而`int&& arg`就是右值引用
6. 示例代码
	```cpp
	#include <iostream>
	#include <utility>
	using namespace std;
	
	// 说明下面这个函数不是一个固定类型，而是一个函数模板
	template<typename T>  // T是一个类型占位符，真正类型由编译器推导
	void wrapper(T&& arg) {  // 转发函数wrapper
	    process(std::forward<T>(arg));   // 完美转发，目标函数process
	}
	
	void process(int& x)  { cout << "处理左值: " << x << endl; }
	void process(int&& x) { cout << "处理右值: " << x << endl; }
	
	int main() {
	    int x = 10;
	    wrapper(x);          // 左值
	    wrapper(20);         // 右值
	
	    int& lref = x;
	    wrapper(lref);       // 左值引用变量
	
	    int&& rref = std::move(x);
	    wrapper(rref);       // 注意：rref本身是左值
	    return 0;
	}
	```

# `C++`如何实现一个单例模式？
1. 单例模式：指的是一个类在整个程序运行期间只允许存在一个实例，并提供一个全局访问点。
2. `C++`实现单例模式需要满足以下几点要求：
	（1）私有化构造函数：将类的构造函数定义为私有，防止外部通过`new`关键字创建多个实例。
	（2）静态实例：在类内部体提供一个静态私有实例，这个实例作为整个程序的唯一实例。
	（3）公有访问方法：提供一个公有的静态方法，通常成为`getInstance`，用于获取类的唯一实例。
	（4）删除拷贝构造函数和赋值操作符：为了防止通过拷贝或赋值来创建新的实例。
3. 单例模式有饿汉式和懒汉式两种实现。
4. 饿汉式
   （1）饿汉式：类实例在类被加载时就进行创建。也就是在进入`main()`函数之前，单例对象就已经准备好了。
   （2）示例代码

	```cpp
	#include <iostream>
	using namespace std;
	
	class Singleton {
	private:
		// 静态成员实例
	    static Singleton instance;
	    
	    // 构造函数私有化
	    Singleton() {
	        cout << "构造 Singleton" << endl;
	    }
		
		// 删除拷贝构造函数和赋值操作运算符
	    Singleton(const Singleton&) = delete;
	    Singleton& operator=(const Singleton&) = delete;
	
	public:
		// 提供一个公有静态方法，用来拿到全局唯一对象
	    static Singleton& getInstance() {
	        return instance;
	    }
	    
	    // 功能函数
	    void doSomething() {
			cout << "doSomething()" << endl;
		}
	};
	
	// 类外定义，真正创建实例
	Singleton Singleton::instance;
	
	int main() {
	    Singleton& s = Singleton::getInstance();
	    return 0;
	}
	```
5. 懒汉式
   （1）懒汉式：第一次真正需要时，才创建对象。
   （2）与饿汉式的区别：
	   `static Singleton instance`这个静态变量定义在静态公有访问函数内部。
	   它的特点是，不会在程序一开始就创建，而是第一次调用`getInstance()`时从创建，后面再调用就直接返回这同一个对象。
	   一句话总结区别：唯一实例放在哪以及什么时候创建。
   （3）示例代码

	```cpp
	#include <iostream>
	using namespace std;
	
	class Singleton {
	private:
	    Singleton() {
	        cout << "构造 Singleton\n";
	    }
	
	    Singleton(const Singleton&) = delete;
	    Singleton& operator=(const Singleton&) = delete;
	
	public:
	    static Singleton& getInstance() {
	        static Singleton instance;   // 第一次调用时才创建
	        return instance;
	    }
	
	    void doSomething() {
	        cout << "doSomething()" << endl;
	    }
	};
	
	int main() {
	    Singleton::getInstance().doSomething();
	    return 0;
	}
	```
6. 线程安全
   懒汉式存在线程安全问题，如果多个线程同时第一次调用获取实例接口，就可能重复创建多个对象。解决方法有：1. 在`getInstance()`函数中加锁； 2. 使用双重检查锁减少不必要的加锁开销； 3. 使用函数内局部静态变量（如上例代码）。这是因为`C++`11保证了**局部静态变量的初始化是线程安全的**。

## 补充知识
### 静态成员函数
1. 静态成员函数：不依赖具体对象，而是属于整个类。
2. 静态成员函数能/不能访问什么？
	（1）不能直接访问非静态成员变量
	（2）不能直接调用非静态成员函数
	（3）可以访问静态成员变量
	```
	// x是某个对象的成员，但func()没有对象，没有this，它不知道访问哪个对象的x
	class A {
	public:
	    int x = 10;
	
	    static void func() {
	        cout << x << endl;   // 错
	    }
	};
	
	// show()是普通成员函数，依赖对象；而func()没有对象
	class A {
	public:
	    void show() {
	        cout << "show" << endl;
	    }
	
	    static void func() {
	        show();   // 错
	    }
	};
	
	// 静态成员变量也是类级别，不属于某个具体对象
	#include <iostream>
	using namespace std;
	
	class A {
	public:
	    static int cnt;
	
	    static void func() {
	        cout << cnt << endl;
	    }
	};
	
	int A::cnt = 100;
	
	int main() {
	    A::func();
	}
	```
