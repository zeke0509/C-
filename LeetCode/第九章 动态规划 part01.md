# [509. 斐波那契数](https://leetcode.cn/problems/fibonacci-number/description/)
## 动态规划引入
凡是遇到递归问题都画个递归树
![[Pasted image 20250901221622.png]]
递归算法的时间复杂度 = 子问题个数 x 解决一个子问题需要的时间 = $2^n * 1$ = O($2^n$) 
观察递归树，我们发现这是因为有些数据被计算了多次，如f(18)被计算了两次
这也就是我们所说的**重叠子问题**
- 备忘录：每次算出的子问题答案先别着急返回，而是先记到备忘录中再返回；如果发现之前已经解决过这个问题了，直接把答案拿出来用。
  一般使用**数组**（或者哈希表）来充当这个备忘录
## 动态规划解法
如果直接再递归中使用备忘录，那么是自顶向下的过程；但是动态规划是实现**自底向上**的过程，即从`f(0) f(1)`入手
- 状态转移方程：描述问题结构的数学形式
  `f(n)` = n, n =0,1;`
  `     = f(n-1) + f(n-2), n > 1`
按照上面的状态方程维护一个`dp table`数组
```
class Solution {

public:

    int fib(int n) {

        if (n == 0 || n == 1) return n;

        vector<int> dp(n + 1, 0);

        dp[1] = 1;

        for (int i = 2; i < n + 1; i++) {

            dp[i] = dp[i - 1] + dp[i - 2];

        }

        return dp[n];

    }

};
```

## 动态规划解法再优化
状态压缩：当我们发现每次状态转移都只需要`DP table`的一小部分，那么就可以尝试用状态压缩来缩小`DP table`的大小。
我们根据状态转移方程，得知当前状态只和之前的两个状态有关，所以并不需要那么长的一个`DPtable`来存储所有状态，只想办法存储之前的两个状态就行了。
```
class Solution {

public:

    int fib(int n) {

        if (n == 0 || n == 1) return n;

        int prev = 0;

        int cur = 1;

        for (int i = 2; i < n + 1; i++) {

            int temp = cur;

            cur += prev;

            prev = temp;

        }

        return cur;

    }

};
```
