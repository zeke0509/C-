# 203.移除链表元素(不必二刷)
## 一刷总结
1. 注意点：
   - 删除链表节点，我们一定要找到前驱节点并操作。这也就说明我们要定义一个虚拟头节点（dummy_head），来操作头节点。
   - 遍历的时候一定要重新定义一个节点，如果直接用头节点来遍历，会改变头节点的值，导致再也找不到真实的头节点了。
   - 我原来认为：要分情况返回空链表和非空链表的情况
	```
	if (dummy->next != nullptr) return dummy->next;
	else return nullptr;
	```
	其实不用这样，直接```return dummy->next``` 就可以解决。因为如果真的都删除掉，那么dummy->next就会等于nullptr，而不需要再进行额外的操作。
   - 虚拟头节基于栈或堆创建都可以
     - 栈：```ListNode dummy(0, head)``` // 这个创建的是一个对象，而不是指针（不咋好用）
       这个表示：ListNode声明dummy是一个ListNode结构体类型，那么dummy就可以调用构造函数```ListNode(int x, ListNode *next)```。这句指令实现了在栈上分配一个ListNode类型的dummy虚拟头节点，无需手动释放。
       使用过程需要有如下注意事项
		```
		ListNode dummy(0, head);   // 栈上对象
		ListNode* cur = &dummy;    // 需要取地址符
		// 其余逻辑一样，无需 delete dummy
		return dummy.next;         // 访问对象的属性需要用点符号
		```
     - 堆：ListNode* dummy = new ListNode(0, head)   // 这个是在堆内存上
       这个创建了虚拟头节点dummy在堆内存上，使用完需要手动释放。
   - 注意构造函数本身是不返回任何东西的，它只是在堆上分配内存空间。
   - 那为什么```ListNode* dummy = new ListNode(0, head);``` 可以返回指针呢？
     ListNode()是一个构造函数：初始化一个ListNode对象，没有返回值
     new的作用：调用构造函数初始化，在堆上分配内存，返回一个指向该对象的指针
   - 被删除的链表的节点需要用delete释放。同理，若不需要虚拟头节点，那么虚拟头节点也要释放

# 707. 设计链表
题目链接：https://leetcode.cn/problems/design-linked-list/description/
## 一刷总结
1. 需要注意的地方
   - 定义了两个私有成员变量：```_dummyHead、_size```，一般习惯将private中的变量前加下划线
   - 类中定义的变量使用小驼峰，如：```_dummyHead、newNode```
   - 进行增删操作的时候，我们要找到操作位置的前驱节点。那么如何判断while()中的条件呢？我们仅需要找个特例即可，如：```while(index--)```，假设index为0，看看是不是我们想要的结果。
   - 跳出函数用：```return;```
   - 插入操作，要先判断是否合理，再`new`，避免内存泄漏。

# 206. 反转链表
题目链接：https://leetcode.cn/problems/reverse-linked-list/solutions/
## 一刷总结
### 双指针方法
1. 算法思路：两个指针分别记录prev与cur指针，因为修改cur的next指针域会造成链表断链，所以引入第三个指针来暂存next指针
### 递归方法
1. 递归：函数调用函数自身，本质是将问题拆解成更小规模的相同问题。
2. 只需三步（我们在写函数之前，先假定这个函数可以解决某类问题，当我们写完函数时，逻辑就自洽了）
   - 确定问题：即，我们写的递归函数要解决什么问题？
     函数应该解决的问题：传入prev和cur节点，然后让cur指向prev
     现在我就假设我们写的函数已经解决了这个问题，不要陷入先有鸡先有蛋
   - 解决基准问题：即，当满足什么条件的时候，函数可以直接返回值？
     当前指针cur如果指向了原链表中尾指针的后继（nullptr），那么可以返回prev（新链表的头节点）
   - 拆解问题：即，拆解成子问题，这个子问题可以实现：每解决成功子问题能够使该问题更接近基准问题。
     子问题如下：让当前指针的指针域指向prev，并分别更新当前指针、前驱指针向后移动一位。
### 递归方法升级版

1. 有关递归的一些自己理解：
   拆解问题的过程和上面其实是一样的。下面记录一些想法：
   递归递的过程是：函数每次会调用函数自己，归就是调用完之后进行一些额外的计算，以达到我们想要的结果
	   在代码中的体现：以f(x)为例
	   f(x)内部出现了f函数的语句就是递的过程，f函数语句之后就是归的过程
	   本质上来讲：
	   递是拆解成子问题的过程
	   归是来体现我们函数的功能（即解决子问题）
2. 以反转链表为例
   把反转链表问题拆解成：反转当前节点及其后继节点
   - 第一步：先假设我们的reverseList能够实现什么功能——反转当前节点及其后继节点，并希望能够返回原链表尾节点指针
     ```head->next->next = head; head->next = null```
   - 第二步：开始递的过程，即函数调用函数自己
     ```reverseList(head->next)```
   - 第三步：设计返回值
     我们其实是希望函数能够返回，我们if语句执行后拿到的那个值。所以返回值的归设计就变为：记录if语句的返回值，并且保存下来，让每次函数归的过程都返回这个值。
     ```cur = reverseList(head->next); 并且retrun cur```
