# 运算符及（循环）语句
## 循环
### for循环
1. 语法：```for(初始化语句；条件判断；操作) { // 代码块 }```
   初始化语句只有开始阶段执行一次。

### range-based for 循环
1. 语法：```for (auto x : something)```
   - something：可以是标准容器（如，vector、set、map等）、原生数组
   - auto x：auto表示它自己推断数据类型，x是something中的遍历元素的拷贝，修改x不能修改something中的值
2. 若想修改something中的元素需要：```for (auto &x : v) x = 2;```

### do while循环
1. 语法：```do { // 循环体（代码块） } while (条件判断);```
   先执行循环体的代码块，然后再检查条件判断。
### break退出循环
1. 语法：break;
2. 作用：终止离它最近的while、do while、for语句。break之后的代码都不会执行。
### continue 退出当前迭代
1. 语法：continue;
2. 作用：和break一样只能出现在for、while和do while循环内部，退出当前循环迭代的剩余部分，然后继续下一次迭代。
   e.g.
			while(std::cin >> a >> b){
	
		        if (a == 0 && b == 0) continue;
	
		        std::cout << a+b << std::endl;
		    }

- 示例一：+运算符重载

	```
	#include <iostream>
	using namespace std;
	
	struct MyNumber {
	public:
	    int val;
	    MyNumber(int x): val(x) {}  // 构造函数
	
	    // 定义一个MyNumber数据类型的内部函数
	    // 函数名为operator+
	    // 函数返回值为MyNumer类型

	    }

	    MyNumber a(3), b(4); // 初始化结构体
	    MyNumber c = a + b;
	    cout << c.val << endl;
	    
	    return 0;
	}
	```

	    return 0;
	}
	
	// 输出返回 GREEN = 1
	```

## 链表
### 前言
1. 线性表包含：==顺序表==和==链表==
   线性表概念：由n（n≥0）个数据特性相同的元素构成的有限序列。
	   - 每个元素有一个直接前驱（除第一个元素）和一个直接后继（除最后一个元素）
	   - 元素个数为 n >= 0；当n=0时，称为空表
   与数组进行辨析：
	   - 线性表要求逻辑上连续，不要求物理内存连续。
2. 顺序表概念（下面的示例基于C，详情见笔记：逊哥）
   用一组连续的内存单元一次存储线性表的各个元素。即，逻辑上相邻的元素，实际的物理存储空间也是连续的。
   所以，数组是顺序表的一种（因为元素存储在连续内存空间上）
   - 顺序表初始化示例
	```
	#include <stdio.h>
	#include <stdlib.h>
	#define MAXSIZE 100          // 注意这里没有分号
	typedef int ElemType;        // 将int类型记作ElemType，以便后续批量更改
	typedef struct {
	    ElemType data[MAXSIZE];  // 定义一个结构体，存放int数组 和 int length
	    int length;
	} SeqList;                   // 记作SeqList数据类型
	
	void initList(SeqList * L) { // 只用于初始化SeqList类型（自定义结构体）中的length
	    L -> length = 0;
	}
	
	int main(void){
	    SeqList list;
	    initList(&list);
	    printf("SeqList的长度为：%d\n", list.length);
	    printf("目前所占用的内存为：%zu字节\n", sizeof(list.data));
	    
	    return 0;
	}
	```
   - 在尾部添加元素
	```
	int appendElem(SeqList * L, ElemType e) {
	    if (L -> length >= MAXSIZE) {          // length 存放当前数据表内容个数
	        printf("顺序表已满\n");
	        return 0;                         // 表示添加失败
	    }
	    L -> data[L -> length] = e;           // 往data[length]存放e
	    L -> length++;
	    
	    return 1;                             // 表示添加成功
	}
	```
   - 遍历顺序表
	```
	void listElem(SeqList * L) {
	    for (int i = 0; i < L -> length; i++) {
	        printf("%d ", L -> data[i]);
	    }
	    printf("\n");
	
	}
	```
3. 链表基本概念：
   - 链表的存储可以是连续，也可以不连续（不同于数组、字符串）
   - 每个数据存储：本身的信息（data数据域）+ 下一个元素的地址信息（next指针域）
   - 链表的第一个节点的存储位置被称为头指针，然后通过next指针域找到下一个节点，直到找到最后一个节点；最后一个节点的next指针域为“空”（用null表示这个空指针）
4. 链表几个概念辨析：
   - 头节点（head）：通常有，是一个（data+next）的结构体。链表的入口节点，不一定存储实际数据（如，虚拟头节点就不存储有效数据，next域指向真正的头节点）
   - 头指针：必须有的，用于访问链表的入口。是链表外部变量，指向头节点的指针。
### 基本操作
1. 定义链表节点：使用结构体（解除传统定义变量方式只能使用一种数据类型的局限）
	```
	struct ListNode {
	    int val;
	    ListNode* next;
	};
	```
	理解：我实际上是定义了一个名为“ListNode”的节点，我可以把这个由结构体定义的ListNode想成一个新的数据类型（类比于int、char）。这个节点（结构体）中存放两部分数据：val 和 next。其中val为数据域的值，next 是一个 指向ListNode结构体（定义的数据类型）的指针。
	
	本质：一个由结构体定义的ListNode是链表中的一个节点。
2. 链表结构体的构造函数（栈）初始化（不常用）：
   - 完整代码

	    ListNode *next;
	    ListNode(int x): val(x), next(nullptr) {} // nullptr表示空指针
	};
	
	int main() {
		// 在栈上创建node
	    ListNode node(42);  // 创建ListNode类型变量 node；再调用初始化列表，传入参数42

	}
	```
3. 链表结构体的构造函数（堆）初始化（常用）：
   更改栈初始化中的代码：```ListNode node(42); ```为```ListNode *node = new ListNode(42);```
   分析：前者是创建node对象在栈上；后者是返回指向node的指针，指向堆上的对象
4. 初始化虚拟头节点
   - 代码实现
	```
	#include <iostream>
	using namespace std;
	  
	struct listNode {       // 定义结构体：int、next；并添加构造函数
	    int val;
	    listNode *next;
	    listNode(int x): val(x), next(nullptr) {}  // 初始化val = x，next=nullptr
	};
	
	// linkedList类来管理链表
	class linkedList {
	private:
	    listNode *head;     // 仅仅是声明，并没有赋值
	    listNode *tail;
	    
	public:
	    // 构造函数方式初始化一个list，并定义虚拟头节点（val = 0；next = nullptr）
	    linkedList(): head(new listNode(0)), tail(head) {}
	}
	```

	linkedList() {

	     tail = head;            // 尾节点就是头节点
	}
	```

   - 实现步骤：

     （3）更新尾节点tail = 新节点指针 newNode

	// pos为要找到的位置。初始化将p指向虚拟头节点，然后进入while循环分析
	// 当pos 为 pos的时候，经过了while循环的p指向了第一个节点
	// 这就说明当pos--更新到1的时候，p指向了pos位置处的节点（想要的）

        return 1;
    }

## 结构体
### 前言
1. 定义：结构体（struct）：C++中一种==自定义数据类型==，可以用来将多个不同类型的变量组合在一起，用来描述一个”对象“或”实体“的多个属性。
2. 与C的区别
   ==C++中结构体的本质是类==，它可以有：
   - 构造函数（constructor）
   - 成员函数
   - 析构函数（destructor）
   - 访问权限（public/private）
   C中结构体仅是数据的打包容器（仅含变量），没有函数，没有构造机制，必须手动赋值、拷贝、释放资源等。
### 定义结构体
1. 基本语法定义方式
	```
	struct Student {
	    string name;
	    int age;
	    float score;
	};
	```
### 初始化结构体
1. 基本初始化方式：逐个成员赋值
	```
	Student s1;
    s1.name = "Alice";
    s1.age = 20;
    s1.score = 89.9;
	```
2. 使用构造函数初始化
   - 在基本语法定义方式的结构体内，添加构造函数
   - 构造函数的特点
     和结构体名相同
     没有返回类型
     不能像成员函数一样自己调用，在构建对象的时候会自动调用
     对象的长相，必须和构造函数一致（参数个数一致）
     一般用于：给数据成员初始化
   - 以具体例子分析构造函数的流程
	```
	struct Point {
	    int x, y;
	    
	    Point(int a, int b) {
	        x = a;
	        y = b;
	    }
	};
	
	int main() {
	    Point p(3, 5);  // ← 看起来很简单的一行，编译器做了很多事！
	}
	```
	我们从Point p(3, 5);这句入手，分析如下：
	- 为对象p分配内存：编译器在栈上开辟空间，大小为```sizeof(Point)```。这类似于：```void *p = alloca(sizeof(Point))```；这句代码仅仅是在栈上开辟内存，不同于C中malloc会返回指针类型```Point *```
	- 自动生成构造函数调用代码：（类似于下面过程）``` call Point::Point(&p, 3, 5)```，Point::Point是在Point结构体作用域下的Point构造函数
	- C++会把p(3,5) 传入 带有隐藏指针的Point(int a, int b)构造函数中（```Point::Point(Point* this, int a, int b)```
	- 构造函数内部赋值：```this->x = a; this->y = b;
	- 构造完成，程序继续执行
3. 初始化列表
   - 作用：用于在对象创建时直接初始化成员变量
   - 语法：```ClassName(参数列表): 成员1(值1), 成员2(值2), ... { 构造函数体（可留空）}```
     语法中不用加分号
### 调用结构体中的成员函数
1. C++中的结构体（struct）本质就是一个轻量级的类（class），所以调用结构体的成员函数操作与调用类的成员函数（方法）差不多
2. 对象：是类的具体实例
3. 成员函数调用的2种方式

   - 对象为指针类型（使用箭头操作符）：```ptr->print();```
   注：区分两种方式。对象指针仅占用一个指针大小，是指向对象的地址；而对象的占据其数据类型对应的内存，是变量（实际内存中的实体）

# 哈希表
## 基础知识
1. 哈希表（Hash table）：根据关键码的值 而 直接访问的 数据结构。
   数组就是一张哈希表：关键码为数组的索引下标；然后通过下标直接访问数组中的元素
2. 解决问题：一般用于快速判断一个元素是否出现在集合里
3. 哈希函数：以一个已知的学生姓名列表为例，如下图所示
   ![[Pasted image 20250706221021.png]]
   哈希函数：将学生的姓名直接映射到哈希表上
	   自变量：一个整型。由HashCode计算得出
	   因变量：哈希表中的位置下标
4. 哈希碰撞：列表中不同元素映射到了哈希表中相同的位置。哈希碰撞的两种常用解决办法
   - 拉链法：发生冲突的元素都被存储在链表中，哈希表中存放链表的头指针。
     关键：选择适当的哈希表大小
   - 线性探针法：依靠哈希表中的空位来解决碰撞问题。（保证tableSize大于dataSize）
     具体做法：假设小王、小李发生哈希碰撞，若冲突的位置先放置了小李，那么就向下找一个位置放置小王的信息。
## 常见的三种哈希结构
### 数组
1. 适用：
   - 键值是整数且范围较小的情况，如0~1000
   - 存在性哈希：判断某些整数是否出现过
   - 计数哈希：统计每个数字出现次数
2. 存在性哈希
   判断某个数字是否出现过
   - 算法思路
     用一个数组visted作为哈希表
     viseted的下标索引为原数组中的值，内部存储true或false
   - 代码实现
	```
	#include <iostream>
	using namespace std;
	
	int main() {
	    bool visited[1001] = {false};  // 假设数的范围是 0~1000
	    int arr[] = {2, 5, 100, 999};  // 输入数组
	    int n = sizeof(arr)/sizeof(arr[0]);
	    // 标记出现过的数字
	    for (int i = 0; i < n; i++) {
	        visited[arr[i]] = true;
	    }
	    // 查询
	    int x;
	    cout << "请输入要查询的数字：";
	    cin >> x;
	    if (visited[x]) cout << x << " 出现过" << endl;
	    else cout << x << " 没出现过" << endl;
	    return 0;
	}
	```
3. 计数哈希
   - 解题思路（以计数26个小写字母出现的次数为例）
     使用一个```int count[26]```的数组，作为哈希表
     哈希函数设计：即找到每个小写字母到```count[index]```中的index的映射关系，```index = s[i] - 'a';```（从字母 -> 索引）
     遍历字符串s，对每个字符，做```count[s[i] - 'a']++;```
   - 若还需找到最大计数，可以记住下面固定模板：
     不断更新已经出现过的最大频率 maxFreq 和 最大频率对应的字符（哈希函数的反函数）
	```
	int maxFreq = -1;
	char result;
	for(int i = 0; i < 26; i++) {
		if (count[i] > maxFreq) {
			maxFreq = count[i];
			result = i + 'a';
		}
	}
	```

# 栈和队列
## 栈
### 基础知识
1. 栈（stack）：限定仅在==表尾（栈顶）==进行插入或删除操作的线性表（所以分为顺序表和链式表，两种存储结构）。即，先进后出
2. 表头端称为栈底，表尾端称为栈顶。如S = (a1, a2, ..., an)，则称a1为栈底元素，an为栈顶元素。
3. 对栈的基本操作：进栈（push）和出栈（pop）。不关注遍历循环、指定位置插入删除（因为我们只操作栈顶）
4. 栈是容器吗？
   栈是容器适配器（container adapter），而不是容器。栈是以底层容器完成其所有工作，屏蔽实现细节，只对外提供统一的接口，底层容器是可插拔的（也就是我们可以控制使用哪种容器来实现栈的功能，即后进先出，如链表、vector数组等）
5. 栈的底层实现：vector、list、deque（双端队列）
   在SGI STL（一种C++标准库）中，默认以deque为缺省情况下栈的底层实现
### 栈的顺序结构实现
1. 结构体内部：数据+top（int类型）
   top：栈顶指针
2. 定义结构体、初始化、进栈、出栈、获取栈顶元素的完整代码
	```
	#include <iostream>
	#define MAXSIZE 100
	using namespace std;
	using ElemType = int;
	
	struct Stack{
	    ElemType data[MAXSIZE];
	    int top;

### 栈的链式结构实现

   - 完整代码
	```
	#include <iostream>
	#define MAXSIZE 100
	using namespace std;
	using ElemType = int;
	
	struct Stack {
	    ElemType data;
	    Stack* next;
	};
	
	// 初始化
	Stack* initStack() {
	    Stack* s = new Stack;
	    s->data = 0;
	    s->next = nullptr;
	    return s;
	}
	
	// 判断是否为空
	int isEmpty(Stack* s) {
	    if (s->next == nullptr) {
	        cout << "空的" << endl;
	        return 1;
	    } else return 0;
	}
	
	// 进栈（头插法）
	int push(Stack* s, ElemType e) {
	    Stack* p = new Stack;   // 堆内存上开辟一个Stack类型空间，用于存放新元素
	    p->data = e;
	    p->next = s->next;      // p的指针域指向原来的头节点
	    s->next = p;            // s本身是指向头节点的指针，而并不是头节点
	    return 1;
	}
	
	// 出栈
	int pop(Stack* s, ElemType* e) {    // e用来接受被删除元素的地址
	    if (s->next == nullptr) {
	        cout << "空的" << endl;
	        return 0;
	    }
	    *e = s->next->data;
	    Stack* q = s->next;     // q用于暂存头元素指针，便于后续释放
	    s->next = q->next;
	    delete q;
	    return 1;
	}
	
	// 获取栈顶元素
	int getTop(Stack* s, ElemType* e) {
	    if (s->next == nullptr) {
	        cout << "空的" << endl;
	        return 0;
	    }
	    *e = s->next->data;
	    return 1;
	}
	
	int main() {
		Stack *s = initStack();
		push(s, 10);
		push(s, 20);
		push(s, 30);
		
		ElemType e;
		pop(s, &e);
	    cout << e << endl;
	
		getTop(s, &e);
	    cout << e << endl;
		
		return 0;
	}
	```

## 队列
### 基础知识
1. 队列：==先进先出的线性表==。它只允许在表的一端进行插入，而在另一端删除。
   队尾（rear）：允许插入一端
   队头（front）：允许删除的一端
   假设队列q = (a1, a2, ... , an)，那么a1是队头，an是队尾元素。
   队列中元素是按照a1,a2,...,an的顺序进入的；那么只有在a1,...,an-1全都离开队列后，an才能退出队列。

### 队列的顺序结构实现
1. 定义结构体：数据（```data[MAXSIZE], ElemType类型```）、队头（front，int类型）、队尾（rear，int类型）
2. 初始化：队头队尾都给0
   需要注意：front内的数据就是队头第一个元素的索引，而rear内的数据其实是队列中最后一个元素的下一位。如上图所示
3. 判断队列是否为空：判断rear是否等于front
4. 出队：```返回data[队头]并让front自加1```
5. 入队：
   需要两层判断，先判断队尾所在的位置是否被占用？
   若被占用，再判断队列是否满了（若未满，则向队头整体移动队列）
   入队的操作如下：
   把入队的元素赋值给```data[队尾]```，并让队尾rear自加1
   注：我们发现入队的操作，移动队列的时间复杂度有点高，所以这就引入了循环队列。
6. 判断队列是否满了：队尾满了则调整队列返回1，若满了返回0
   若front>0，说明可以移动：
   找到移动的步长，for循环挨个移动元素；更新front为0，rear = rear-步长
7. 完整代码
	```
	#include <iostream>
	#define MAXSIZE 100
	using namespace std;
	using ElemType = int;
	
	struct Queue{
	    ElemType data[MAXSIZE];
	    int front;
	    int rear;
	};
	
	// 队列初始化
	void initQueue(Queue* Q) {
	    Q->front = 0;
	    Q->rear = 0;
	}
	  
	// 判断队列是否为空
	int isEmpty(Queue* Q) {
	    if (Q->front == Q->rear) {
	        cout << "空了" << endl;
	        return 1;
	    } else return 0;
	}
	  
	// 队尾满了则调整队列返回1，若满了返回0
	int queueFull(Queue* Q) {
	    if (Q->front > 0) {
	        int step = Q->front;
	        for (int i = Q->front; i <= Q->rear; i++) {
	            Q->data[i-step] = Q->data[i];
	        }
	        Q->front = 0;
	        Q->rear = Q->rear - step;
	        return 1;
	    } else {
	        cout << "真满了" << endl;
	        return 0;
	    }
	}
	
	// 出队
	ElemType dequeue(Queue* Q) {
	    if (Q->front == Q->rear) {
	        cout << "空了" << endl;
	        return 0;
	    }
	    ElemType e = Q->data[Q->front];
	    Q->front++;
	
	    return e;
	}
	  
	// 入队
	int equeue(Queue* Q, ElemType e) {
	    if (Q->rear >= MAXSIZE) {   // 队尾在的位置已经到头了
	        if (!queueFull(0)) return 0;
	    }
	    Q->data[Q->rear] = e;
	    Q->rear++;
	
	    return 1;
	}
	
	// 获取队头数据
	int getHead(Queue* Q, ElemType* e) {
	    if (Q->front == Q->rear) {
	        cout << "队头为空" << endl;
	        return 0;
	    }
	    *e = Q->data[Q->front];
	    return 1;
	}
	  
	int main() {
	    Queue q;
	    initQueue(&q);
	    equeue(&q, 10);
	    equeue(&q, 20);
	    equeue(&q, 30);
	    equeue(&q, 40);
	    equeue(&q, 50);
	    cout << dequeue(&q) << endl;
	    cout << dequeue(&q) << endl;
	    ElemType e;
	    getHead(&q, &e);
	    cout << e << endl;
	  
	    return 0;
	}
	```

### 循环队列的顺序结构
1. 循环队列：循环队列最大实际容纳元素数量是MAXSIZE-1
   因为我们需要牺牲一个空间位的做法来区分队列的“空”和“满”的状态（front = rear时，无法区分是满了还是空的冲突），分析如下：
   - ==队列已满==：如果尾指针再向前移动一格就会等于头指针，说明已满。
   - ==队空==：如果头指针=尾指针，说明没有元素为空。
   - 涉及到front和rear自增或自减的操作：
     原来是直接```Q->rear++```；现在需要```Q->rear = (Q->rear + 1) % MAXSIZE```
     原来是直接```Q->front++;```现在需要```Q->front = (Q->front + 1) % MAXSIZE```
2. 循环队列为解决：如果队头为空，队尾被占用，需要整体移动队列的问题
3. 定义结构体：与普通队列一致
4. 入队：判断队满条件 和 rear自增操作不同
5. front自增操作不同；还有就是返回值我更改了另一种写法（这条其实可以不改，无所谓）

### 队列的链式结构
1. 队列链式结构剖析：队列内的每个元素都是一个节点。通过两个结构体实现
   结构体Queue：存放队列的==队头节点指针和队尾节点指针==
   结构体QueueNode：队列节点。存放data域和next域
   下文的头节点不存放元素（类似于虚拟头节点），链表的首元素为队头，尾元素为队尾（可颠倒）
2. 结构体声明（两部分）
   一个队列节点内，拥有 ==元素数据(data域)== 和 ==指向一下元素的指针(next域)==
   一个队列结构体内，拥有队头节点和队尾节点。（以队头节点为例，里面存放了队头的元素和指针，只不过队头是虚拟头节点，不用来存放元素，其data域为0）
   这样定义的==原因==：有关队列的操作，我们只关注头尾，不会对队列中间元素进行操作。那么，我们通过一个**队列结构体记录头尾节点**，便于我们做入队和出队的操作。再用一个**节点的结构体，实现队列内的元素链式连接**。
	   - 代码实现
		```
		struct QueueNode {
		    ElemType data;
		    QueueNode* next;
		};
		
		struct Queue {
		    QueueNode* front;
		    QueueNode* rear;
		};
		```
	   - 注意事项
	     并未使用data[MAXSIZE],这是顺序结构才有的。链式结构体每个节点只存一个元素
3. 队列链式结构的初始化（有虚拟头节点）：初始化两个结构体
   初始化队列结构体，内部用于存放虚拟头节点和尾节点。因为初始情况头节点和尾节点应该都存0并指向nullptr，所以按照如下要求初始化头节点和尾节点
   初始化队列节点结构体，堆内存上分配一块QueueNode类型的区域（data域为0，next域为空）
	```
	Queue* initQueue() {
	    Queue* q = new Queue;
	    QueueNode* node = new QueueNode;
	    node->data = 0;
	    node->next = nullptr;
	    q->front = node;
	    q->rear = node;
	    return q;
	}
	```
4. 判断队列是否为空
   头节点和尾节点指针相同
	```
	int isEmpty(Queue* q) {
	    if (q->front == q->rear) return 1;
	    else return 0;
	}
	```
5. 入队（尾插法）
   让原尾节点的指针域 = 新插入元素指针
   更新记录队列尾节点 = 新插入元素指针
	```
	void equeue(Queue* q, ElemType e) {
	    QueueNode* node = new QueueNode;
	    node->data = e;
	    node->next = nullptr;
	    q->rear->next = node; //让原来队列的尾节点指针域为新插入元素指针
	    q->rear = node;       // 更新记录队列q的尾节点
	}
	```
6. 出队（释放首元素，即头节点的下一个节点）
   需要注意，要加一个判断：删除的节点是否为尾节点。原因如下：
   正常来讲，出队操作不会更改尾节点（q->rear），但是当出现情况“删除的元素正好为原来的rear”，这时我们需要更新rear=front来表示队列为空了。如若不更新，rear仍指向被释放的节点
	```
	int dequeue(Queue* q, ElemType* e) {
	    QueueNode* node = q->front->next;   // node为首元素指针
	    *e = node->data;
	    q->front->next = node->next;        // 令头节点的指针域 = 删除节点的下一个节点
	    if (q->rear == node) q->rear = q->front;
	    delete node;
	    return 1;
	}
	```
7. 获取头元素
	```
	ElemType getFront(Queue* q) {
	    if (isEmpty(q)) {
	        cout << "空的" << endl;
	        return 0;
	    }
	    return q->front->next->data;
	}
	```
8. 完整代码
	```
	#include <iostream>
	#define MAXSIZE 100
	using namespace std;
	using ElemType = int;
	
	struct QueueNode {
	    ElemType data;
	    QueueNode* next;
	};
	
	struct Queue {
	    QueueNode* front;
	    QueueNode* rear;
	};
	
	// 初始化
	Queue* initQueue() {
	    Queue* q = new Queue;
	    QueueNode* node = new QueueNode;
	    node->data = 0;
	    node->next = nullptr;
	    q->front = node;
	    q->rear = node;
	    return q;
	}
	
	// 判断队列是否为空
	int isEmpty(Queue* q) {
	    if (q->front == q->rear) return 1;
	    else return 0;
	}
	
	// 入队
	void equeue(Queue* q, ElemType e) {
	    QueueNode* node = new QueueNode;
	    node->data = e;
	    node->next = nullptr;
	    q->rear->next = node; //让原来队列的尾节点指针域为新插入元素指针
	    q->rear = node;       // 更新记录队列q的尾节点
	}
	
	// 出队
	int dequeue(Queue* q, ElemType* e) {
	    QueueNode* node = q->front->next;   // node为首元素指针
	    *e = node->data;
	    q->front->next = node->next;        // 令头节点的指针域 = 删除节点的下一个节点
	    if (q->rear == node) q->rear = q->front;
	    delete node;
	    return 1;
	}
	
	// 获取队头元素
	ElemType getFront(Queue* q) {
	    if (isEmpty(q)) {
	        cout << "空的" << endl;
	        return 0;
	    }
	    return q->front->next->data;
	}

		equeue(q, 20);
		equeue(q, 30);
		equeue(q, 40);
		equeue(q, 50);
		ElemType e;
		dequeue(q, &e);
	    cout << "出队" << e << endl;
		dequeue(q, &e);
	    cout << "出队" << e << endl;
	    cout << getFront(q) << endl;
		return 0;
	}

### 优先级队列
1. 优先级队列```priority_queue```是C++STL中提供的一个容器适配器，属于标准数据结构之一。
2. 声明方式
	   - 引入头文件：```#include <queue>```
	   - 声明语法：```priority_queue<T, Container, Compare>  // 注意是<>而不是()```
	     T ：队列中存储元素的数据类型
	     Container：底层容器类型，必须支持随机访问（vector为默认）。这里Container必须是一个能够包含元素T的类型，如```vector<T>```
	     Compare：是一个拥有函数调用符（即```()```）的类型，且必须返回bool类型。
	     实现元素的比较方式，决定“被优先弹出”规则。具体说明：当compare实现两个数a和b的比较，在优先级队列内部，就是比较前者（靠近队首）、后者（靠近队尾）两个数的大小。若compare为true，代表后者优先级高；反之，前者优先级高。
	     如```less<int>```：当前者<后者为true，所以优先弹出最大值
	     如```greater<int>```：当前者>后者为true，优先弹出后者（最小值）
	   - 常用比较规则的头文件
	     默认情况就是大顶堆。
	     若要使用小顶堆，需要```greater<int>```比较规则，这个规则定义在头文件```#include <functional>```中   
3. 自定义Compare
   Compare类型：拥有```operator()```，且必须返回bool结果。
   这就需要我们对括号运算符进行重载。
   示例一：
	```
	#include <iostream>
	#include <utility>
	using namespace std;
	
	struct CompareBySecond {
		bool operator()(const pair<int, int>& a, const pair<int, int>& b) {
			return a.second > b.second;
		}
	};
	
	int main() {
		pair<int, int> x = {1, 23};
		pair<int, int> y = {2, 32};
		CompareBySecond cmp;
		if (cmp(x, y)) cout << "x.second > y.second" << endl;
		else cout << "x.second <= y.second" << endl;
		
		return 0;
	}
	```
4. 举例
	   - 声明优先级队列
	     ```
	     // 声明大顶堆maxHeap
	     #include <queue>
	     priority_queue<int> maxHeap;  
	     // 等价于
	     priority_queue<int, vector<int>, less<int>> maxHeap;
	     // 因为底层容器默认vector，优先弹出规则默认为“大数先弹出”
	      
	      // 声明小顶堆minHeap
	      #include <queue>
	      #include <functional>
	      priority_queue<int, vector<int>, greater<int>> minHeap;
	     ```
5. 常用方法
   优先级队列是广义队列，所以自然就不会去用队列的方法。如```q.front(); q.back();```；而是用栈的方法。
	   - 访问队列中优先级最高的元素：```q.top();```
	   - 蒋元素val插入队列：```q.push();```

## 一些应用
### 根据后缀表达式求值
- 中缀表达式：```5+6*3```
  后缀表达式：563*+
- 计算后缀表达式
  遍历字符串：遇到数字就进栈，遇到操作符就连续弹出两个数字出栈，分别放到操作符的右侧和左侧，计算后的结果再进栈。
- 完整代码
	```
	#include <iostream>
	#include <stack>
	using namespace std;
	  
	// 判断字符串中的字符是什么类型的东西
	enum class ContentType {
	    LEFT_PARE, RIGHT_PARE,
	    ADD, SUB, MUL, DIV, MOD,
	    EOS, NUM
	};
	
	char expr[] = "82/2+56*-";  // 以'\0'结尾（即EOS）
	
	// 用于获取字符数组[index]处的字符类型
	// 并为传入的 symbol变量 赋值为字符数组[index]
	ContentType getToken(char* symbol, int* index) {
	    *symbol = expr[*index];
	    (*index)++;   // 更新index，获取字符数组下一个元素
	    // 蒋字符类型的符号，转换为枚举类型
	    switch(*symbol) {
	        case '(':
	            return ContentType::LEFT_PARE;
	        case ')':
	            return ContentType::RIGHT_PARE;
	        case '+':
	            return ContentType::ADD;
	        case '-':
	            return ContentType::SUB;
	        case '*':
	            return ContentType::MUL;
	        case '/':
	            return ContentType::DIV;
	        case '%':
	            return ContentType::MOD;
	        case '\0':
	            return ContentType::EOS;
	        default:
	            return ContentType::NUM;
	    }
	}
	
	// 传入一个栈，并在函数内打印结算结果
	int eval(stack<int>* s) {
	    char symbol;     // 用于装遍历的字符
	    int op1, op2;       // 用于op1 symbol op2
	    int index = 0;
	    ContentType token;  // 用于存放枚举类型
	    token = getToken(&symbol, &index); 
	    while(token != ContentType::EOS) {
	        if (token == ContentType::NUM) (*s).push(symbol - '0');
	        else {
	            op2 = (*s).top();
	            (*s).pop();
	            op1 = (*s).top();
	            (*s).pop();
	  
	            switch(token) {
	                case ContentType::ADD:
	                    (*s).push(op1 + op2);
	                    break;
	                case ContentType::SUB:
	                    (*s).push(op1 - op2);
	                    break;
	                case ContentType::MUL:
	                    (*s).push(op1 * op2);
	                    break;
	                case ContentType::DIV:
	                    (*s).push(op1 / op2);
	                    break;
	                case ContentType::MOD:
	                    (*s).push(op1 % op2);
	                    break;
	                default: 
	                    break;
	            }
	        }
	        token = getToken(&symbol, &index);
	    }
	    cout << (*s).top() << endl;
	    (*s).pop();
	    return 1;
	}
	
	int main() {
	    stack<int> s;
	    eval(&s);       // 传入栈的地址
	
	    return 0;
	}
	```
### 中缀表达式转后缀表达式
- 难点1：后缀表达式的符号是按照中缀的优先级排列的，后缀表达式每个符号前的两个输出参与计算（包括新计算的结果重新入栈）
  重新观察中缀与后缀表达式：中缀表达式为```5+6*3```，转换成后缀表达式为563*+
  分析：因为我们先算乘法，所以乘号在前。
- 难点2：对于括号的处理
  中缀表达式：x / (i - j) * y  →  后缀表达式：x i j - / y *
- 算法思路：
	  - （与后缀表达式求值相反）如果扫描到操作数，直接输出；如果是字符的话，做判断。
	  - 准备一个栈，栈内预存放运算符的优先级。起始在栈底先放个0
	  - 对于运算符是否压栈的判断流程：
	    若新运算符的优先级==大于==栈顶元素优先级，==压入栈==中
	    若新运算符的优先级 <= 栈顶元素，持续将栈顶元素出栈输出，直至新运算符优先级＞栈顶元素优先级，然后将新运算符压入栈中
	    当左括号在栈外时，属于最高优先级；当左括号在栈里时，属于最低优先级。
		    这就需要我定义两个数组，来定义符号的优先级
	    若新运算符是右括号，且栈顶元素不是左括号，持续出栈并输出，直到栈顶为左括号出栈结束，最后将左括号也出栈（但不输出）
	    最后将栈中元素依次输出（不包括栈底的0） 
- 代码实现
	```
	#include <iostream>
	#include <stack>
	using namespace std;
	
	// 判断字符串中的字符是什么类型的东西
	enum class ContentType {
	    LEFT_PARE, RIGHT_PARE,
	    ADD, SUB, MUL, DIV, MOD,
	    EOS, NUM
	};
	
	char expr[] = "x/(i-j)*y";
	
	// 用于获取字符串[index]处的字符类型
	ContentType getToken(char* symbol, int* index) {
	    *symbol = expr[*index];
	    (*index)++;
	    switch(*symbol) {
	        case '(':
	            return ContentType::LEFT_PARE;
	        case ')':
	            return ContentType::RIGHT_PARE;
	        case '+':
	            return ContentType::ADD;
	        case '-':
	            return ContentType::SUB;
	        case '*':
	            return ContentType::MUL;
	        case '/':
	            return ContentType::DIV;
	        case '%':
	            return ContentType::MOD;
	        case '\0':
	            return ContentType::EOS;
	        default:
	            return ContentType::NUM;
	    }
	
	}
	
	int print_token(ContentType token)
	{
	    switch (token)
	    {
	        case ContentType::ADD:
	            cout << "+"; break;
	        case ContentType::SUB:
	            cout << "-"; break;
	        case ContentType::MUL:
	            cout << "*"; break;
	        case ContentType::DIV:
	            cout << "/"; break;
	        case ContentType::MOD:
	            cout << "%"; break;
	        default:
	            return 0;
	    }
	    return 1;
	}
	
	void postfix(stack<int>* s) {
	    // in_stack中存放的是枚举元素的栈内优先级，与枚举的索引一一对应
	    // 如：我发现一个加号ContentType::ADD，它的值就是2
	    // 这时我去in_stack[2] = 12 即加号的优先级
		int in_stack[] = {0, 19, 12, 12, 13, 13, 13, 0};
	
	    // out_stack存放的是枚举元素的栈外优先级，只有左括号是不同的。
		int out_stack[] = {20, 19, 12, 12, 13, 13, 13, 0};
	
	    ContentType token;
	    int index = 0;
	    s->push(static_cast<int>(ContentType::EOS));
	    char symbol;
	    token = getToken(&symbol, &index);      // 返回token为枚举中的数据类型；symbol为当前遍历的符号
	    
	    do {
	        if (token == ContentType::NUM) cout << symbol;
	        else if (token == ContentType::RIGHT_PARE) {
	            // 如遇到新运算符为右括号，那么输出栈内左括号之前的所有元素
	            // 并把左括号出栈但不输出
	            while (s->top() != static_cast<int>(ContentType::LEFT_PARE)) {
	                print_token(static_cast<ContentType>(s->top()));
	                s->pop();
	            }
	            s->pop();       // 左括号出栈但不输出
	        // 如果遇到除右括号以外的其它运算符号
	        } else {
	            while (in_stack[s->top()] >= out_stack[static_cast<int>(token)]) {
	                print_token(static_cast<ContentType>(s->top()));
	                s->pop();
	            }
	            s->push(static_cast<int>(token));   // 压入新元素的优先级
	        }
	        token = getToken(&symbol, &index);
	    } while(token != ContentType::EOS);
	    
	    // 字符串结束的时候，要把栈内残留的符号输出
	    while (s->top() != static_cast<int>(ContentType::EOS)) {
	        print_token(static_cast<ContentType>(s->top()));
	        s->pop();
	    }
	    cout << endl;
	}
	
	int main() {
	    stack<int> s;
	    cout << expr << endl;
	    postfix(&s);
	
	    return 0;
	}
	```

# 树
## 树与二叉树
### 基础知识
1. 树是一个或多个节点的有限集合。
   存在一个称为根的特定节点，其余的节点被分为n个互不相交的集合T1、T2、...、Tn，其中每个集合都是一棵树。T1、T2、...、Tn称为根节点的子树。
2. 一些定义
	   - 结点：树中的一个独立单元
	   - 结点的度：结点拥有的子树数
	   - 树的度：树内各个结点度的最大值
	   - 叶子：度为0的结点（或叫终端结点）
	   - 非终端结点：度不为0的结点
	   - 双亲和孩子：结点的子树的根称为该结点的孩子。相应地，该结点称为孩子的双亲
	   - 层次：结点的层次从根开始定义，根为第一层，根的孩子为第二层，以此类推
	   - 高度：树的层数
### 树的基本性质
![[Pasted image 20250723113710.png]]
- 性质一：树中所有结点数 = 所有结点的度数之和 + 1
- 性质二：对于度为m的树，第i层上最多有```m^(i-1)```个结点
  模拟理解：第一层为1个，第二层最多1 x m个，第三层最多1 x m x m个
- 性质三：对于高度为h的，度为m的树，最多有```(m^h - 1) / (m - 1)```个结点。
  计算过程：由性质2的等比数列求和

### 二叉树基础知识
1. 二叉树（Binary Tree）定义：n(n >= 0 )个结点所构成的集合。它或为空树（n = 0)，或为非空树
   对于非空树T，有如下性质：
   （1）有且仅有一个称为根的结点
   （2）除根节点外的其余结点可以分为两个不相交的子集，称为T的左子树和右子树
   （3）二叉树的子树有左右之分，其次序不能任意颠倒
2. 基本形态
   ![[Pasted image 20250723234505.png]]

### 二叉树的性质
- ==性质一==：二叉树的第i层最多有```2^(i-1) （i>=1)```个结点
- ==性质二==：深度为k的二叉树最多有```2^k-1 （k>=1)```个结点
- ==性质三==：对于任何非空的二叉树T，如果叶子结点的个数为n0，而度为2的结点数为n2，则n0 = n2 + 1

- 判断题目：
  （1）![[Pasted image 20250724000717.png]]
  不是完全二叉树。
  （2）![[Pasted image 20250724001157.png]]
  是完全二叉树
- 性质四：具有n个结点的完全二叉树的深度为$(\log_2 n) + 1$（向下取整）
- 性质五：如果对一棵有 $n$ 个结点的完全二叉树（其深度为 $\lfloor \log_2 n + 1 \rfloor$）的结点按层序编号（从第 1 层到第 $\lfloor \log_2 n + 1 \rfloor$ 层，每层从左到右），则对任一结点 $i$（$1 \leq i \leq n$），以下结论成立：

1. 如果 $i = 1$，则结点 $i$ 是二叉树的根，无双亲；如果结点 $i > 1$ 则其双亲是结点 $\lfloor i/2 \rfloor$（向下取整）

2. 如果 $2i > n$，则结点 $i$ 无左孩子（结点 $i$ 为叶子结点）；否则其左孩子是结点 $2i$

3. 如果 $2i + 1 > n$，则结点 $i$ 无右孩子；否则其右孩子是结点 $2i + 1$
- ==性质五==：完全二叉树度为1的结点，只能有1个（总结点数为偶数）或者0个（总结点数为奇数）

## 二叉树的存储结构 —— 链表结构
### 二叉树遍历——递归
二叉树的搜索分为：深度优先搜索和广度优先搜索
（1）深度优先搜索（DFS）：沿着一条分支尽可能“深入”到叶子，再回退到最近的分叉继续，直到所有结点都被访问。它通常用递归或栈（也可也用迭代法，能用递归就一定能用迭代法）来实现。如，前序、中序、后序遍历。
（2）广度优先搜索（BFS）：从根开始，按层从上到下，从左到右依次访问节点。它通常用队列迭代法实现，先入先出。如，层序遍历

- 根据遍历结果推导二叉树
  例：已知一颗二叉树的前序遍历序列为ABCDEF，中序遍历序列为CBAEDF，请问这颗二叉树的后续遍历结果？
  答：根据前序序列知道A是根节点，根据中序序列（左根右）可知A的左右两部分分别为（CB）、（EDF）
  根据前序序列AB知，B是A的左孩子
  根据中序序列CB知，C是B的左孩子
  根据前序序列CD知，D是A的右孩子
  根据前序序列DE，并不能知道E是D左孩子还是右孩子；
  根据中序序列ED知，E是D的左孩子
  根据中序序列DF知，F是D的右孩子
  所以二叉树为：
     A
    B       D
  C       E  F
  所以后续遍历结果：C B E F D A
- 二叉树遍历性质
  已知前序遍历和中序遍历，可以唯一确定一颗二叉树
  已知中序遍历和后序遍历，可以唯一确定一颗二叉树
  已知前序序列和后序遍历，是不能确定一颗二叉树的
## 线索二叉树
### 前言
1. 通过遍历可以得到二叉树的线性排列，但这样线性序列只有在遍历时才能得到。
   容易误区的地方：把二叉树的左右孩子误当作根节点的后继节点，想象成了链表。
   二叉树中这种所谓前驱和后继关系是由我们遍历方式决定的，三种不同的遍历方式，得到的顺序是不同的。
2. 线索二叉树用于解决：想要得到二叉树的线性排列，就只能通过递归遍历的弊端。实现像遍历链表一样轻松（一层while循环就能写完）的遍历
3. 线索化：利用叶节点的空余空间记录前驱、后继（要事先先定好遍历方式）
4. 线索二叉树的初概览
   我们先确定以中序遍历的结果存储链表结构，所以HDIBJEAFCG就是我们要求链表的指向
   将线索指针加入到每个有空余空间的节点（空余空间指的是该节点的lchild或rchild）
   ![[Pasted image 20250725160937.png]]
   ==节点总数为n的二叉树一定有n+1个空==，解释如下：
   可以将上述的结构存储在含有n个结点的二叉链表中。对于一个二叉链表，由三部分组成：Data域（存放节点信息）、lchild指针（指向左孩子节点）、rchild指针（指向右孩子节点）。
   那么该链表总共有2n个链域，但实际上，只有n-1条链域被使用，因为n个节点的二叉树有n-1条边，因此会存在n+1个空指针 
5. 那如何区分所存的地址表示的 是孩子节点 还是线索呢？
   改造原来结构体中的lchild和rchild
   ![[Pasted image 20250725161510.png]]
### 线索二叉树 —— （以中序遍历为例）循环链表的构建
- 头节点的```lchild```指向二叉树的根，头节点是自己造的
- 头节点的```rchild```指向遍历的最后一个节点（线索）
- 第一个节点（遍历的第一个节点）的```lchild```指向头节点
- 最后一个节点（遍历的最后一个节点）的```rchild```指向头节点
- 图示上述过程
	  ![[Pasted image 20250725162607.png]]
- 代码实现
  1. 创建线索二叉树的结构体
	每个节点包含：数据域+左右孩子指针+线索标志位
	```
	typedef char ElemType;
	// 创建线索二叉树节点数据类型（结构体）
	struct TreadNode {
	    ElemType data;
	    TreadNode* lchild;
	    TreadNode* rchild;
	    int ltag;
	    int rtag;
	};
	typedef TreadNode* ThreadTree;
	```
  2. 初始化线索二叉树（递归实现）
	通过先序方式把str内容构造进线索二叉树，此部分只设置正经孩子的线索标志位。即只设置```ltag/rtag = 0```，未设置的后续会被线索标志位填充
	```
	// 初始化线索二叉树
	void createTree(ThreadTree* T) {
	    ElemType ch = str[idx++];
	    if (ch == '#') *T = nullptr;
	    else {
	        *T = new TreadNode;
	        (*T)->data = ch;
	        createTree(&(*T)->lchild);
	        if ((*T)->lchild != nullptr) (*T)->ltag = 0;
	        createTree(&(*T)->rchild);
	        if ((*T)->rchild != nullptr) (*T)->rtag = 0;
	    }
	}
	```
  3. 开始线索化：
	创建一个头节点```head```，将其左指针指向根，右指针指向中序尾节点（构成闭环）
	调用```threading()```进行线索化
	参数说明：因为我仅仅创建并修改头节点，所以要传递头节点的指针。而线索化不用创建新节点，所以传递ThreadTree 即可。
	```
	void inOrderTreading(ThreadTree* head, ThreadTree T) {
	    *head = new ThreadNode; // 自己创建一个头节点
	    (*head)->ltag = 0;      // 头节点的lchild指向正经孩子节点
	    (*head)->rtag = 1;      // 头节点的rchild指向遍历的最后一个节点
	    (*head)->rchild = *head;// 初始先将右孩子指向自己
	    // 注意：这里头节点的右孩子不能为空，因为后面会访问prev(head)的右孩子
	    // 若右孩子为空，则给它加后继。但是头节点的右孩子不需加后继
	  
	    // 当线索二叉树为空时，头节点的左右孩子都指向自己
	    if (T == nullptr) (*head)->lchild = *head;
	    else {
	        (*head)->lchild = T;    // 头节点的lchild指向二叉树的根
	        prev_ = (*head);        // 去记录最后一个节点
	        threading(T);
	        // 最后一个节点线索化
	        prev_->rchild = *head;
	        prev_->rtag = 1;
		
	        (*head)->rchild = prev_;    // 头节点的rchild指向遍历的最后一个节点
	    }
	}
	```
  4. 具体线索化
	整体是一个递归的过程，直到threading(T)中参数T为空，什么都不返回直接弹出，开始归的过程，即从上一次rhreading(T)，此时T->lchild为空开始归
	注意如下顺序：
		- 通过threading(T->lchild)一直递归找到最左孩子，也就是第一个节点。此时T为第一个节点，```T->lchild```是为空的，那么把```T->lchild```线索化，让它指向prev
		- 当找到了第一个节点后，我们需要更新prev了，但是在更新prev前，一定要判断prev的右孩子是否为空，即是否能够被用来存后继节点。若为空，则令prev = T
		  思考：为什么要判断prev的右孩子，而不是像找左孩子一样去判断T的左孩子呢？
		  这是因为：右孩子放的是后继节点，我们没有办法预知下一个节点到底是谁，所以在prev更新前，去初始化一下prev的右孩子。
		- 要更新prev了：prev = T
		- 再递归去找右节点（左根右）
	```
	void threading(ThreadTree T) {
    // 当递归到T为空的时候，直接退出函数，执行上一层
	    if (T != nullptr) {
	        threading(T->lchild);
	        if (T->lchild == nullptr) {
	            T->ltag = 1;
	            T->lchild = prev_;
	        }
	        if (prev_->rchild == nullptr) {
	            prev_->rtag = 1;
	            prev_->rchild = T;
	        }
	        prev_ = T;
	        threading(T->rchild);
	    }
	}
	```
  5. 使用线索进行中序遍历
	遍历函数中，只需要传入我们定义的虚拟头节点head就行了，它包含了所有信息。
	中序线索二叉树遍历的本质（两步）：
		找到中序结果序列中的第一个节点 -> 然后依次找节点的后继，直至其后继为空
		- 找第一个节点：
		  从根节点一直找左子树，直至最底层的左子树就是中虚结果序列的第一个节点
		          while (curr->ltag == 0) curr = curr->lchild;
		        cout << curr->data << ' ';
		- 依次找节点的后继
		  若该结点rtag=1，则右链为线索，直接指示其后继
		  若该结点rtag=0，遍历找 该结点右子树 的 最底层左子树（类似于：把这个右子树节点看作子根节点，找该节点的第一个节点）
		          while (curr->rtag == 1 && curr->rchild != T) {
		            curr = curr->rchild;
		            cout << curr->data << ' ';
		        }
		        curr = curr->rchild;
		        再执行第一步
	```
	void inOrder(ThreadTree T) {
	    ThreadTree curr;
	    curr = T->lchild;
	    while (curr != T) {     // 当curr等于头节点，退出循环
	        while (curr->ltag == 0) curr = curr->lchild;
	        cout << curr->data << ' ';
	  
	        while (curr->rtag == 1 && curr->rchild != T) {
	            curr = curr->rchild;
	            cout << curr->data << ' ';
	        }
	        curr = curr->rchild;
	    }
	    cout << endl;
	}
	```

	#include <string>
	using namespace std;
	
	typedef char ElemType;
	
	// 创建线索二叉树节点数据类型（结构体）
	struct ThreadNode {
	    ElemType data;
	    ThreadNode* lchild;
	    ThreadNode* rchild;
	    int ltag;
	    int rtag;
	};
	
	typedef ThreadNode* ThreadTree;
	void threading(ThreadTree T);
	string str = "ABDH##I##EJ###CF##G##";
	int idx = 0;
	ThreadTree prev_;       // 用于记录上一次访问的节点
	
	// 初始化线索二叉树
	void createTree(ThreadTree* T) {
	    ElemType ch = str[idx++];
	    if (ch == '#') *T = nullptr;
	    else {
	        *T = new ThreadNode;
	        (*T)->data = ch;
	        createTree(&(*T)->lchild);
	        if ((*T)->lchild != nullptr) (*T)->ltag = 0;
	        createTree(&(*T)->rchild);
	        if ((*T)->rchild != nullptr) (*T)->rtag = 0;
	    }
	}
	
	// 开始（初始化头节点并更改遍历结果第一个节点和最后一个节点的指向）
	void inOrderTreading(ThreadTree* head, ThreadTree T) {
	    *head = new ThreadNode; // 自己创建一个头节点
	    (*head)->ltag = 0;      // 头节点的lchild指向正经孩子节点
	    (*head)->rtag = 1;      // 头节点的rchild指向遍历的最后一个节点
	    (*head)->rchild = *head;// 初始先将右孩子指向自己
	
	    // 当线索二叉树为空时，头节点的左右孩子都指向自己
	    if (T == nullptr) (*head)->lchild = *head;
	    else {
	        (*head)->lchild = T;    // 头节点的lchild指向二叉树的根
	        prev_ = (*head);        // 去记录最后一个节点
	        threading(T);
	        // 最后一个节点线索化
	        prev_->rchild = *head;
	        prev_->rtag = 1;
	        (*head)->rchild = prev_;    // 头节点的rchild指向遍历的最后一个节点
	    }
	}
	
	void threading(ThreadTree T) {
	    // 当递归到T为空的时候，直接退出函数，执行上一层
	    if (T != nullptr) {
	        threading(T->lchild);
	        if (T->lchild == nullptr) {
	            T->ltag = 1;
	            T->lchild = prev_;
	        }
	        if (prev_->rchild == nullptr) {
	            prev_->rtag = 1;
	            prev_->rchild = T;
	        }
	        prev_ = T;
	        threading(T->rchild);
	    }
	}
	
	// 基于线索的遍历
	void inOrder(ThreadTree T) {
	    ThreadTree curr;
	    curr = T->lchild;
	    while (curr != T) {     // 当curr等于头节点，退出循环
	        while (curr->ltag == 0) curr = curr->lchild;
	        cout << curr->data << ' ';
	
	        while (curr->rtag == 1 && curr->rchild != T) {
	            curr = curr->rchild;
	            cout << curr->data << ' ';
	        }
	        curr = curr->rchild;
	    }
	    cout << endl;
	}
	
	int main() {
	    ThreadTree head;
	    ThreadTree T;
	    createTree(&T);
	    inOrderTreading(&head, T);
	    inOrder(head);
	}
	```

3. 将节点N1替换刚刚取出的两个节点，并加入到有序序列

## 类型可擦除的可调用包装器：`std::function`
### 有关类和结构体的进一步理解
- C++中`struct`和`class`在语法层面上几乎等价，都可以定义成员变量、成员函数、构造函数、析构函数、继承、多态等功能。唯一区别是**默认访问权限和继承权限**不同。
  `struct`的默认权限是`public`
  `class`的默认权限是`private`
- 那么如果想访问内部成员变量/成员函数，用`::`还是`.`？
  静态成员（只存一份，所有对象共享）：用`::`
  普通成员：用`.`
  类空间中的成员函数：用`::`，如`Class::成员函数`
  e.g.
	```
	class B {
	public:
		static int count;
		int val;
		B(int v) : val(v) { count++; }
	};
	int B::count = 0;
	int main() {
		B b1(10);
		B b2(20);
		cout << b1.count << endl; // 2, (可运行，不推荐）实际上是 B::count
		cout << B::count << endl; // 2, 推荐写法
		cout << b1.val << endl; // 10, 访问普通成员
		// cout << b1::val << endl; 错误写法
	}
	```
- 成员函数与普通函数的指针
  对于普通函数，函数名会自动退化成函数指针；
  对于成员函数，不会发生这种隐式转换，而**必须使用`&Class::Member`来获取成员函数指针**
- 普通函数指针类型 vs 成员函数指针类型
  ![[Pasted image 20250820204320.png]]
  说明：`void (A::*)(int)`表示一个指针，指向属于A类的成员函数，这个成员函数返回`void`，参数为`int`；而并不特指A中某个成员函数
  拆分理解：
	  - `A::*`：指向A的成员
	  - `(int)`：参数是`int`类型
	  - `void`：返回类型是`void`
  例如，有成员函数如下
	```
	struct A {
	    void bar(int);
	};
	```
	那么，`void(A::*)(int)` 是指针，可看作一种数据类型；`&A::bar`是成员函数的指针，是一个具体的值（等到有实例，才能真正去用），它的类型正好是`void(A::*)(int)`
### `std::function<(Args...)>`
1. 头文件：`#include <functional>`
2. 可装下的东西
   能装下并调用任何可调用对象。
   这里可调用对象指它满足：能以参数`Args..`被调用并且其返回值可转换成`R`，包括：
	   - 普通函数/函数指针、
	   - 成员函数指针（需要配合对象或`std::bind / lambda`）、
	   - 函数对象（有`operator()`的类/结构体）、
	   - `lambda`
2. 基本语法：
   `std::function<返回类型(参数列表)> f;`
3. 举例
	- 封装函数指针
	  普通函数的函数名自动退化成**函数指针**，即我们要封装的内容就是把`mulitpy`封进`func1`
		```
		double multiply(double a, double b) {
			return a * b;	
		}
		int main() {
			// mutiply 函数名表示函数指针
			std::function<double(double,double)> func1 = multiply;
			double res = func1(1.1, 2.3);
			cout << res << endl;
			return 0;
		}
		```
	- 封装类/结构体的成员函数
	  `&类名::成员函数名`是成员函数指针，理解为：取 成员函数的 地址我们要封装的就是**成员函数指针**
	  基本语法的参数列表中，第一个参数为类名的引用（如`Linear&`），从第二个参数开始时成员函数的参数类型
		```
		#include <functional>
		#include <iostream>
		using namespace std;
		struct Linear {
			Linear(float k, float b): k_(k), b_(b) {} // 构造函数
			float f(float x) {return k_ * x + b_;} // 成员函数
			float k_, b_; // 成员变量
		};
		int main() {
			std::function<float(Linear&, float)> mf = &Linear::f;
			Linear l(1.2, 2.3);
			float res = mf(l,5);
			cout << res << endl;
			return 0;
		}
		```
	- 封装类/结构体的成员变量
	  类似于封装成员函数，返回类型为成员变量的类型，参数列表中第一个参数为类名的引用，没有第二个参数（因为成员变量不接收参数）

		struct Linear {
			Linear(float k, float b): k_(k), b_(b) {} // 构造函数
			float f(float x) {return k_ * x + b_;} // 成员函数
			float k_, b_; // 成员变量
		};
		int main() {
			std::function<float(Linear&, float)> mf = &Linear::f;
			Linear l(1.2, 2.3);
			float res = mf(l,5);

			std::function<float(Linear&)> k = &Linear::k_;
			cout << k(l) << endl;

## 绑定可调用对象与实参：`bind()`
### `std::bind(F&& f, Args&&..)`
1. 头文件：`#include <functional>
2. 功能：把**可调用对象**与**部分/全部**实参绑定，返回一个新的可调用对象
   若想绑定部分实参，需使用占位符`std::placeholders::_1`
3. 语法：`bind(F&& f, BoundArg&&... bound_args)`
   `F&& f`：是一个可调用对象，可以是
	   - 普通函数/函数指针、
	   - 成员函数指针、
	   - 函数对象（有`operator()`的类/结构体）、
	   - `lambda`
   `BoundArg&&... bound_args`：要绑定的实参，对应未来调用时函数需要的参数。支持
	- 直接传值、
	- 传占位符`_1, _2, ...`（表示未来传入的第i个参数）、
	- `std::cref(x)`, 按const引用绑定，也就是传入的是x的引用
   返回值是函数对象，可以像普通函数一样调用
4. 举例
	- 简单绑定参数
		```
		#include <functional>
		#include <iostream>
		using namespace std;
		using namespace std::placeholders;
		int sum(int a, int b, int c) {
			return a + b + c;	
		}
		int main() {
			int n = 6;
			auto f = bind(sum, 1, _1, cref(n));
			n = 11;
			int res = f(5);
			cout << res << endl;  // 输出 1 + 5 + 11 = 17
			return 0;
		}
		```
	- 绑定成员函数
	  以源码为例，
	  第一个参数为成员函数指针：`&ChatServer::onConnection`，仅仅表示指向成员函数的**地址**，这个地址的类型是`void(ChatServer::*)(const TcpConnectionPtr&)`，表示指向**属于`ChatServer类`的成员函数，这个成员函数返回`void`类型，接收`TcpConnectionPtr&类型`参数**，不包含对象信息。
	  第二个参数为对象地址`this`，这样才能保证在回调的时候直到哪个对象来调用这个成员函数。
	  第三个对象为成员函数的参数
	  综上，
	  `bind(&ChatServer::onConnection, this, _1)`

## `Lambda`
- 功能：允许在需要的地方定义匿名函数，而不需要按照传统方式定义一个完整的函数。`Lambda`表达式本质就是个结构体。
- 使用场景：
  - API：配合`std::function`能做**回调、任务队列、线程入口**等统一接口
  - 简洁：临时使用一个函数但不想定义一个完整函数
  - 闭包：`Lambda`可以捕获外部变量，形成闭包，使得函数可以访问和操作外部作用域变量
- 语法：`[capture](params) mutable noexcept -> RetType {// body }
	- `capture`（捕获列表）：决定带哪些外部变量、按值还是引用
		- `[]` 不捕获
		- `[x]` 按值捕获 x（复制一份）
		- `[&x]` 按引用捕获 x（外部变化能感知）
		- `[=]` 默认按值捕获所用到的一切
		- `[&]` 默认按引用捕获所用到的一切
		- `[x = expr]` **初始化捕获**（C++14+），可结合 `std::move` 做移动捕获
		- `[this]` 捕获当前对象指针；`[*this]`（C++17）按值捕获对象（拷贝当前对象）
	- `params`（参数列表）：与普通函数相同，也支持 **泛型参数** `auto`（C++14+）
	- `mutable`：允许**修改按值捕获的副本**（不影响外部原值）
	- `noexcept`：承诺不抛异常
	- `-> RetType`：指定返回类型；省略时由编译器推断（分支返回类型不一致时需要显式写）
	- **非捕获 lambda** 可隐式转换为**函数指针**（常用于 C 接口）
- 举例
	```
	#include <iostream>
	#include <functional>
	using namespace std;
	int main() {
	    auto lambda = [](int x) { return x * 2; };
	    std::function<int(int)> f = lambda; // 调用转换构造函数
	    cout << f(5) << endl; // 输出 10
	}
	```
	解析：这里`function<int(int)>`是类型可擦除，表示把类型F是什么擦掉了，只关心后面的对象是否能够用(int)调用并返回int。
	而`lambda`定义为接收参数`int x` 返回 `int x*2`类型，完全是只可以的！
	所以`f(5)`会把5当做`x`传入`lambda`
	注意：这里有关`lambda`的定义中，`()`内的参数就和正常函数的参数没区别，即需要传入。**而`[]`表示的是如果结构体需要的参数不是从`()`传入，那就需要从外部获取，`[]`表明获取方式**

## 多态
### 多态初了解
- 多态：指子类继承了父类的方法，但是有变现出不同的状态
- 语法：
  - 父类中的成员函数要使用虚函数：`virtual 函数返回类型 函数名(参数) {..可以为空.}`
  - 子类的对成员函数重写：`函数返回类型 函数名(参数) override {...}
e.g.1
```
#include <iostream>
using namespace std;

// 父类中使用虚函数定义成员函数
class Animal {
public:
    virtual void walk() { cout << "animal" << endl; }
};

// 方法一：子类对父类虚函数重写
class Dog : public Animal {
public:
    void walk() override { cout << "Dog" << endl; }
};

class Cat : public Animal {
public:
    void walk() override { cout << "Cat" << endl; }
};

// 方法二：把要呈现出多态的虚函数封装在一个函数LetItWalk中
void LetItWalk (Animal& an) { an.walk(); }

int main () {
    Dog dog;
    Cat cat;
    LetItWalk(dog);     // dog
    LetItWalk(cat);     // cat
    dog.walk();         // dog
    cat.walk();         // cat
}
```
### 多态的实现方式
实际上C++会在虚函数父类里添加一个**虚函数表**`vfptr`，用于指向对象实际类型所对应的虚函数地址，如上例中：
```
class Animal {
public:
    virtual void walk() { cout << "animal" << endl; }
};
```
实际上是
```
class Animal {
public:
	void* vfptr;       // 虚函数表
    virtual void walk() {cout << "animal" << endl; }
};
```
### 析构函数为虚函数的情况
- `delete`：先调用析构函数，再释放内存
- 析构函数：
  对于栈上对象（自动存储期）：退出作用域自动掉一次析构（不要`delete`）
  对于堆上对象（动态存储期, `new`得来）：必须`delete ptr;`才会调用析构并释放内存
- 什么情况下**基类析构必须是`virtual`?**
  后续会通过基类指针删除派生对象，则基类析构必须是虚函数，否则只会调用基类的析构函数，而不会调用子类的析构函数
	```
	struct Base { virtual ~Base() = default; virtual void run() = 0; };
	struct D1 : Base { void run() override {/*...*/} };
	struct D2 : Base { void run() override {/*...*/} };
	
	std::unique_ptr<Base> make(int t){
	    if(t==1) return std::make_unique<D1>();
	    else     return std::make_unique<D2>();
	}
	
	auto p = make(1);   // 你甚至不知道返回的是哪个派生类
	p->run();           // 但能多态调用
	```
# 宏
什么是宏？
- 宏由预处理器在编译前做**纯文本替换**
- 宏不做类型检查、没有作用域的概念（只是文本替换）
## 以可变参数宏为例，讲解宏
- 用`__VA_ARGS_`来替代函数宏中的`...`参数（不推荐）
	```
	#define LOG(fmt, ...) printf(fmt, __VA_ARGS__)
	```
	如果`...`为空，会报错
- 用`##__VA_ARGS_`来替代函数宏中的`...`参数（推荐）
  `##`：粘贴操作符，可以有效处理`...`为空的情况
  `#define LOG(fmt, ...) printf(fmt, ##__VA_ARGS__)`
- 用`do {...} while (0)`结构让“多语句宏”看起来像单条语句，多行宏需要用`\`换行
	```
	// LOG_INFO("%s %d", arg1, arg2)
	#define LOG_INFO(logmsgFormat, ...)       \
	    do {                                  \
	        Logger& logger = Logger::instance();      \
	        logger.setLogLevel(INFO);                 \
	        char buf[1024] = {0};                     \
	        snprintf(buf, 1024, logmsgFormat, ##__VA_ARGS__); \
	        logger.log(buf);                          \
	    } while(0)
	```

# 锁、条件变量
## 锁
- `std::lock_guard<std::mutex lock(m_mutex);`
  `lock_guard`是`RAII`封装，构造时自动调用`m_mutex.lock()`，析构时自动调用`m_mutex.unlock()`，确保作用域结束一定释放资源。
  它的声明周期就是锁的生命周期，不能主动`unlock()`
- `std::unique_lock<std::mutex> lock(m_mutex);`
  可以主动`unlock()/lock()`
  能与`std::condition_varibale`配合使用（因为要求必须能`unlock`）
## 条件变量
- `m_condvariable.wait(lock)`
  参数说明
  - `m_condvariable`：一个`std::condition_variable`类型的对象
  - `wait(lock)` ：是`condition_variable`的一个成员函数，参数必须是`std::unique_lock<std::mutex>`类型的对象。
  函数功能
  - 自动释放互斥锁`lock.unlock`
  - 让当前线程进入等待队列（阻塞，即休眠，`cpu`可去执行其他线程），直到`notify_one()或notify_all`唤醒
  - 被唤醒时，`wait()`会先重新获取锁(`lock.lock()`)，再返回
- 举例如下：
	```
	T Pop() {
		std::unique_lock<std::mutex> lock(m_mutex);
		while (m_queue.empty()) {
		// 日志队列为空，线程进入wait状态
		m_condvariable.wait(lock);
		}
		T data = m_queue.front();
		m_queue.pop();
		return data;
	}
	```
	1. 进入 `Pop()`：执行  `std::unique_lock<std::mutex> lock(m_mutex);`  → 获得互斥锁。
	2. 判断 `while (m_queue.empty())`：
	   如果为空：执行 `m_cond.wait(lock);`，`wait` 会**原子地释放这个同一个 `lock` 持有的互斥量**（即`lock.unlock`）并让线程睡眠。
	3. 被 `notify_one()` 唤醒时：  `wait` **先把同一个互斥量重新加锁**（仍然通过同一个 `lock` 对象），然后 `wait` 返回到那一行的下一步。
	4. 立刻**重新判断** `while (m_queue.empty())`：
	   如果还是空，继续 `wait(lock)`（释放→睡眠→被唤醒→加锁…）；
	   如果不空，**跳出 `while`**，往下继续执行
- 好处：
  唤醒后马上加锁，避免多个线程同时访问共享资源
