# 进程和线程之间有什么区别
1. 定义
	- 进程：是一个运行中程序的实例，也是操作系统**资源分配的最小单位**。进程里包含一个或多个线程
	- 线程：是程序执行的最小单元
2. 资源
	- 进程：有自己独立的文件描述符、地址空间，包含**代码段、数据段、堆、栈（栈是由每个线程单独分配）**
	- 线程：共享进程的资源，如**代码段、数据段、堆**，但会维护自己的**栈、寄存器、线程控制块**
3. 资源开销
	- 进程：需要切换整个上下文，开销大
	- 线程：只需要切换栈和寄存器，开销小，因此线程比进程更轻量
4. 通信
	- 进程：进程间通信需要IPC，常见方式如**管道、消息队列、共享内存**等
	- 线程：由于线程共享同一进程的内存空间，所以它们可以直接读写数据，但需要枷锁避免资源竞争。
5. 安全性
	- 进程：一个进程崩溃，不会直接影响别的进程（隔离性好）。如打开浏览器，一个标签页对应一个进程，关闭一个标签页不会影响其他标签页。
	- 线程：一个线程的崩溃可能会影响整个线程的稳定性。如在`word`文档里，不同线程可以共享同一个文件资源，分别负责排版渲染、拼写检查、后台保存等，假如拼写检查线程出错，可能会导致整个程序卡死
6. 多进程 or 多进程 选择
	- 有安全、隔离性需求：多进程
	- 性能优化，需要共享数据：多线程

# 并行和并发有什么区别
并发是指相同的时间段内处理多个任务，这些任务不是同时发生的，而是交替执行的。通常依靠任务调度和时间片轮转，即使在单核CPU上也能实现。
并行是同一时刻执行多个任务，这些任务可以同时进行，通常依赖多个处理单元（如，多个CPU核心），每个核心执行一个独立任务，实现性能提升。

# 用户态和内核态的区别
用户态和内核态是操作系统为保护系统资源和实现权限控制而设计的两种不同的**CPU运行级别**。
用户态是普通应用程序运行的状态，权限受限，不能直接操作硬件，也不能随意访问内核数据结构。
内核态是操作系统内核运行的状态，具有最高的权限，可以访问所有硬件资源、执行所有CPU指令，管理进程、内存文件系统等。
内核态和用户态的切换会主要发生在系统调用、异常和中断三种情况。
系统调用：用户态的应用程序要请求内核服务，如读取文件、分配内存、创建进程等，就会通过系统调用进入内核态，操作系统内核处理完请求后，再切换回用户态继续执行应用程序。
异常处理：当用户态应用程序发生异常时，如除0错误或者非法指令，CPU会切换到核心态，让内核的异常处理程序来处理异常。
中断处理：当CPU收到外部设备发出的中断信号时，会切换到内核态执行中断处理程序。执行完相应的中断程序后，会将CPU切换回用户态。

# 介绍一下几种典型的锁
两类底层的锁模式：**互斥锁和自旋锁**，其它锁可以基于这两种锁实现。
对于互斥锁来说，如果在加锁过程中失败，线程会释放CPU资源给其它线程，当前线程进入阻塞状态。
对于自旋锁来说，如果加锁失败，线程会进入**忙等待**，即线程会在尝试获取锁时不断轮询，一直占用CPU资源，直到检测到锁可用。
所以互斥锁是涉及到两次上下文切换成本，第一次是获取锁失败，线程释放CPU资源进入阻塞状态；第二次是获取到锁后，它需要被唤醒，从阻塞态切换回运行状态。而自旋锁是不涉及上下文切换。这两种锁的应用场景不同：对于被锁住代码执行时间很短应该用自旋锁，对于被锁住代码执行时间很长应该用互斥锁。
基于这两种锁，会产生其它锁的形态。
第一种是读写锁，它允许多个线程同时读共享资源，只允许一个线程进行写操作。分别读和写两种状态。这也是悲观锁的**概念**。
悲观锁是指它会认为多线程同时修改共享资源的概率比较高，所以访问共享资源时要上锁。
乐观锁是指会先假设冲突发生的概率低，它会先对共享资源进行修改，修改完后再验证是否有其它线程同时修改，如果有修改再去执行例如放弃操作。
注：
上下文切换：保存和恢复每个线程的运行状态就是上下文切换。

# 解释一下什么是线程安全？
线程安全是指：多个线程访问或操作同一资源时，程序的行为仍然正确，即不会因为多线程的并发而导致数据不一致或程序崩溃。
如何保证线程安全可以从两类思路展开：
第一种是通过同步机制控制访问，常见的技术有：
1. 互斥锁（`Mutex`）
   互斥锁用于保证同一时刻只有一个线程可以访问共享资源。
   `std::mutex`是`C++`标准库中的互斥锁
   `lock_guard`和`unique_lock`可以用于在作用域内自动管理互斥锁的加锁和解锁。
   `lock_guard`：它的内部持有`mutex`引用，构造函数自动调用`lock()`，析构函数调用`unlock`写法简洁，作用域结束自动解锁，避免忘记释放导致死锁。
   `unique_lock`比`lock_guard`更加灵活，提供了额外功能。如：延迟加锁、手动`unlock/lock`多次等功能。
2. 读写锁：
   读写锁允许多读一写，适用于读多写少的场景
3. 条件变量：需要配合互斥锁使用，实现线程等待和通知
4. 信号量：限制并发数量或让多个线程在同一同步点集合。
第二类思路是通过无锁手段减小共享
5. 原子操作：是由硬件保证单步完成，避免数据竞争。如`CAS(Compare-And-Swap)`比较并交换是一种具体的原子操作，CPU在一条指令内完成”比较+交换“两个步骤，对外表现为不可分割，这就是它的原子性。
6. 线程封闭（`thread_local`）：每个线程拥有独立数据，不需要同步。
在我的MPRPC项目中，我采用多Reactor架构，让每个连接固定在一个IO线程，从模型上避免跨线程访问；日志系统使用生产者-消费者队列和条件变量异步写盘，保证了高并发下的线程安全并兼顾性能。

# 进程调度算法，你了解多少？
在批处理（并行处理环境）场景下，最基本的算法是
**先到先服务算法**：先来的进程先执行，直到这个进程执行完或者被阻塞才能执行下一个进程。这种基本算法有一个问题，我们的进程分为两种，一种是计算密集型，需要长时间计算，另一种是IO密集型，它执行的时间短，但是会执行IO操作。所以先到先服务算法就可能被一个长任务长时间占用，后面的短任务即使执行时间很短，它也必须要等待，会影响执行效率，针对这个问题，改进后是
**最短作业优先调度算法**：它会优先选择运行时间最短的线程调度。但是带来的问题是，短任务一直插队，导致长任务一直执行不了。在这个基础上改进后，出现了
**最短剩余时间优先调度算法**：当一个新的作业到达时，它的整个运行时间与当前进程的剩余时间作比较，执行时间更短的任务。
上述这些算法都是偏向长作业或者短作业其中一方，并不会利好另一方的执行。为了平衡长短作业的处理，出现了
**优先级调度算法**：为每个进程分配一个优先级，按优先级进行调度。
在交互式场景下，一般使用
**时间片轮转调度算法**：为每个进程分配一个时间片，进程轮流执行，时间片用完后切换到下一个进程。一般比较好的时间片设定为：20~50ms左右。
**多级队列算法**：它是将时间片轮转调度算法和优先级调度算法相结合，将进程分为不同的优先级队列，每个队列都有自己的调度算法。高优先级队列中的进程会优先被调度，只有在高优先级队列为空才能轮到下一级队列。比如可以让交互性任务（如鼠标控制）始终位于高优先级队列。

# 进程间有哪些通信方式
进程通信`(IPC,Inter-Process Communication)`是操作系统为进程提供在彼此之间传递数据和信号的方法。进程间通信主要有以下几种方式，
**管道（`Pipe`）**：分为匿名管道和命名管道。匿名管道适用于具有父子进程关系的进程间使用，它的生命周期与进程绑定。而命名管道可以用于任意两个进程之间。管道是一种半双工的通信方式，只允许单向的数据传输。
**消息队列**：内核维护了消息的链表，可以设置优先级，允许进程发送和接受消息。
**信号**：用于告知某个进程发生了某个事件。
**信号量**：是一个计数器，可以用来控制多个进程对共享资源的访问，常作为一种锁机制，防止某进程正在访问共享资源时，其它进程也访问该资源。因此主要作为进程间以及同一进程内不同线程之间的同步机制。
**共享内存**：在不同进程的地址空间映射同一块物理内存区域。速度最快，无需在内核和用户空间拷贝。但通常需要配合同步机制（如信号量、互斥锁）避免同时写导致数据竞争。
**Socket套接字**：主要用于在客户端和服务器之间的网络通信。

# 解释一下进程同步和互斥，以及如何实现进程同步和互斥
**进程同步**：是指协调和管理多个并发执行的存在**依赖关系或特定顺序**的进程。
**进程互斥**：是指某一个时刻只允许一个进程访问某个共享资源。当一个进程正在使用共享资源时，其它进程不能访问该资源。
对于互斥来说，有一个很常用的实现是锁，分为互斥锁和读写锁。互斥锁是指每个进程对共享资源操作前都需要先上锁，使用完再释放锁。读写锁是对读操作和写操作进行区分，可以实现读的过程可以允许其它进程读，但是写的过程不允许其它进程读或写。
对于进程互斥还有一个解决方案是信号量，信号量也能解决进程同步的问题。信号量分为P操作和V操作，P操作检查信号量，如果资源可用，就减少计数，然后使用资源；V操作归还资源，增加信号量的计数，并唤醒等待的进程。

# 什么是死锁，如何避免死锁？
死锁是指系统中两个或多个进程在执行过程中，因争夺资源而造成的一种相互等待的现象。具体来说，为了防止多个线程同时访问资源，线程在访问资源前会先给资源加锁，只有当前线程获取到锁后，才能对资源进行操作，否则需要等待其它线程释放锁资源后，它获取到才能对资源操作。
死锁的一个典型场景是，以两个线程为例，他们同时需求对方使用的资源，都在等待对方释放锁，从而进入一个无限等待的过程。
死锁的发生需要同时满足四个条件
1. 互斥条件：线程对资源的访问是互斥的，不可以同时被多个线程访问。
2. 持有并等待：它是指进程已经持有至少一个资源，同时又提出新的资源请求而阻塞。举个例子来说，假设现在有两个线程，线程1和线程2，两个资源，资源A和资源B。线程1在持有资源A的锁的情况下，它还需要获取资源B的锁，那么它在等待资源B的锁过程中，不会释放资源A的锁，这就是持有并等待。
3. 不可抢占：已经分配给线程的资源，在未使用完之前不能被其它线程强行剥夺。
4. 循环等待：所涉及的线程对资源的获取形成了一个环路，形成一个循环等待的关系。
避免死锁就是通过打破上述的四个条件任意一个即可。
对于互斥条件，不是很好打破。针对第二条件，打破持有并等待，让线程在开始时获取它所有使用资源的锁，防止在执行过程中发生等待；第三个条件，打破不可抢占，线程主动释放一些当前已经占用的资源，让其它线程有可能去抢占这个资源。第四个条件，打破循环等待，使用资源有序分配法破坏环路，对资源的申请顺序做出规定，不同的线程请求资源都按相同顺序访问资源，可以避免环路产生。
